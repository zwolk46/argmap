Stream E — Amendment 1 (UI Overhaul)

Amendment to stream_e_visual_polish_v1.html  ·  Session: I.10b UI overhaul  ·  Date: 2026-05-20

Per Article XI § 1, stream documents are immutable once written. This amendment records that the visual layer specified in stream_e_visual_polish_v1.html has been re-skinned on top of shadcn/ui primitives (style radix-maia, base color neutral) plus Phosphor icons, using the user-supplied preset bbVKmki. The structural commitments in stream_e_uiux_v1.html (page topology, named regions, density, interaction patterns, status-symbol semantics) are not modified by this amendment. The domain tokens calibrated in v1 (status palette, mode accents, edge families, severity, sub-flags, AI attribution, motion, z-index) are preserved unchanged. What changes here is the base palette, the icon library, the component library, and the application font.

What changed

1. Component library: shadcn/ui (radix-maia)

The hand-rolled argmap primitive layer (src/ui/primitives/*) was migrated to thin wrappers around shadcn primitives generated by npx shadcn@latest init --preset bbVKmki --template vite. Public exports and prop shapes were preserved where possible so call sites and tests did not have to be touched en masse. Internal implementations now ride on Radix UI primitives (via the radix-ui umbrella package) with cva-driven variants and tailwind-merge class composition.

Consequence: button shape is pill (rounded-4xl in radix-maia) rather than the previous medium-radius rectangle (--radius-md: 6px). Dialog, drawer, tooltip, popover, dropdown, and toast use Radix-portaled DOM rather than always-mounted React children — visible behavior unchanged, but tests that previously walked the container subtree for portaled content were updated to query screen.* or document.body.*.

2. Base palette: neutral OKLCH (preset)

The warm-neutral HSL palette in v1 (hsl(36 16% 97%) canvas, hsl(30 8% 12%) text) is replaced by the preset's pure achromatic neutral, expressed in OKLCH:

The v1 argmap tokens (--color-surface-canvas, --color-text-primary, etc.) continue to exist alongside these. Surfaces that have already been migrated to shadcn read --background/--foreground; surfaces still on the argmap primitive layer (none after the I.10b session) read the v1 names. The v1 names remain defined in tokens.css for any legacy or transitional usage.

Visual effect: a slight loss of warmth, with the canvas reading cleaner and more screen-native. The change is uniform across all base surfaces and can be reversed by editing the :root block in src/index.css.

3. Icon library: UICONS → Phosphor

The previous UICONS CDN dependency (three font-icon CSS files served from cdn-uicons.flaticon.com) is replaced by @phosphor-icons/react bundled with the app. index.html drops the three CDN links. Phosphor weights map cleanly to the previous UICONS families:

  • Phosphor regular ≈ UICONS regular-rounded
  • Phosphor bold ≈ UICONS bold-rounded
  • Phosphor fill ≈ UICONS solid-rounded

The argmap TypeIcon and SeverityIcon components retain their public APIs. Internal glyph mapping moves to Phosphor: Question, Hash, TextT, ChatCircleDots, Flag, Diamond, BookOpen, Circle, TreeStructure, Scales, Check, Warning, X, Minus, Sparkle, Star. A transitional UIcon shim with a slug-to-Phosphor lookup remains in src/ui/primitives/uicon.tsx for surfaces that still reference UICONS slug names; the shim deletes when no callers remain.

4. Font: self-hosted Inter Variable

The Google Fonts CDN link for Inter is replaced by @fontsource-variable/inter (npm dependency, bundled). --font-sans in tokens.css updated to prefer "Inter Variable" with "Inter" retained as a graceful fallback. Source Serif 4 (prose tab) and JetBrains Mono (monospace identifiers) continue to load from Google Fonts CDN — those are not part of the preset.

What did not change

Stream E v1 commitments retained

  • Page topology: Home / Frame Building (three-pane) / Argument Running (two-pane + bottom panel) / Version History (side pane).
  • Density: 32–40px row heights, 4px spacing base, no whitespace indulgence.
  • Named regions: top bar / left pane / canvas / right pane / drawer / bottom panel. Selectors and z-bands preserved.
  • Status symbols: ✓ satisfied, ○ open, ⚠ contested, ✕ foreclosed, — not-applicable. Same glyph + color = same meaning, everywhere.
  • Five-status node treatment (header strip + left rail + body wash + optional strikethrough/hatch). Implementation lives in src/ui/styles/global.css via .canvas-node[data-status][data-kind] selectors. Untouched by this amendment.
  • Domain tokens: status palette (satisfied/open/contested/foreclosed/not_applicable), mode accents (frame-building slate-blue / argument-running burnt-sienna), edge families (structural neutral / foreclosure brick-red / supports teal / contradicts orange / answers purple / annotation gray), severity (pass/warning/error), legal sub-flags (binding navy / persuasive slate), AI attribution (plum), milestone (warm gold). Defined in src/ui/styles/tokens.css, unchanged.
  • Motion scale: instant 50ms / fast 100ms / base 150ms / medium 250ms / slow 350ms / pulse 1800ms; prefers-reduced-motion: reduce collapses all to 0.01ms. Unchanged.
  • Z-index scale (11 bands from canvas=1 to tour=10000). Unchanged.
  • Type scale (10/11/12/13/15/17/22px), spacing scale (4/8/12/16/24/32/40/48px), line heights, letter spacing. Unchanged.

Stream E v1 deferrals still deferred

  • Dark theme. The preset ships a .dark palette block in src/index.css; it is present but not activated. v1 light-only commitment in stream_e_visual_polish_v1.html stands.
  • Responsive / mobile / narrow-screen layouts. Top-bar chips still hide at 720/560/480px breakpoints via global.css, but the three-pane / two-pane workspaces do not reflow.
  • Full WCAG 2.1 AA audit (Stream H/I deferral).
  • Print stylesheet.

Constitutional compliance

  • Article II § 1 (frame/argument split) — UI overhaul does not touch frame or argument-session data flow. Pages, panels, dialogs preserve their dispatch into FrameStore / SessionStore unchanged.
  • Article II § 2 (determinism) — UI layer does not invent or reorder canonical compute output. No new LLM use was introduced; existing AI hooks (useAiSuggestion) retain their interface. [data-status] / [data-kind] attribute semantics on canvas nodes are preserved, so visual status presentation continues to read from canonical compute results.
  • Article II § 3 (per-node satisfaction criteria) — node-frame.tsx and edge renderers untouched in structure; only glyph library swapped.
  • Article II § 4 (independent versioning) — version-history pane re-skinned; underlying repository contracts, version-tree shape, restore + compare flows unchanged.
  • Article II § 5 (human-readable structure) — five-status canvas treatment retained; the readability commitment is the load-bearing visual contract and was the explicit non-negotiable for the canvas re-skin.
  • Article II § 6 (use-case commitment) — legal and general modes both rendered; mode-accent token system carries the legal/argument distinction; legal sub-flag chip (binding/persuasive) preserved on Authority nodes.
  • Article III § 2 (legal practice fit) — radix-maia style is calm, neutral, professional. Pill-shape buttons are softer than the prior medium-radius look but read appropriately for a serious analytical tool. The user authorized the preset's base palette swap explicitly.
  • Article IV § 6 (non-reversal of user direction) — the user authorized option 2a (theme-only, stock shadcn primitives styled with the preset tokens) with the explicit standard "if this option will produce professional results, then continue." The implementation aimed at that standard.
  • Article XI § 1 (stream immutability) — this amendment was produced rather than editing the v1 visual polish document.

Test impact

Test suite at session start: 1756 passing. After all surface migrations + test fixes: 1757 passing / 0 failing across 163 test files. The portal-query failures from the foundation pass (suggestion-drawer, validation-drawer) and one Radix-Tabs-in-happy-dom failure were resolved during integration; the OutputViewTabs component was switched back to a raw-button implementation (Tailwind-styled, role-based) to avoid Radix's pointer-event synthesis dependency.

Most existing tests use role-based and text-based queries and survived unchanged. Test updates were required where:

  • An assertion matched on .argmap-* class names (replaced with role/text queries).
  • An assertion walked the React container subtree for content that is now Radix-portaled to document.body (rewritten as screen.* or document.body.* queries).
  • A confirm flow used role="dialog" where the new ConfirmDialog renders as role="alertdialog" (one chrome test, plus any surface tests doing the same).
  • An IconButton variant assertion used data-variant, which shadcn's own data-variant now occupies — argmap's variant key moved to data-argmap-variant.

Deployment

Per CLAUDE.md, overhaul-scale changes are not auto-deployed to Vercel. The branch will be handed to the user for review and explicit deploy authorization. Vite dev server runs at http://localhost:5173 for local inspection.
