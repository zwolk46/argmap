Cross-document resolution notes (for self-review)

While compiling this spec from stream_i_contracts_v1.html Contract 4, the state spec's action-runner section, the runtime spec's ComputeInput shape, the persistence spec's atomicity contracts, and Stream D's D1–D3 source content, three precision items arose. Each is resolved by this spec in the constitutionally clean direction. They join the F-003 family (the same family as the I.2, I.3, I.4, and I.5 pending items) and are logged as a single in-session-escalated flag for a future I.1 review session to merge into a contracts-doc bump. Downstream coding is not gated on the merge; this spec is canonical for the modes module per Stream I conventions.

  1. Per-action signatures inherited from I.5: synchronous pure transformations. The user's session prompt fixed this directly: "Contract 4's import-direction graph is preserved; only the per-action signatures change (per this session's pending flag)." Contract 4's FrameBuildingActions.createNode(args): Promise<NodeRef> (and the parallel ArgumentRunningActions) is replaced by the dispatch-table entries the state spec pinned (stream_i_state_spec_v1.html "Modes action signatures refined"). This spec implements those tables. The constitutional grounding from I.5 is preserved verbatim and not re-adjudicated: the state layer is the source of truth for save payloads and is the single application caller of compute(); modes-action methods are pure transformations the action runner composes with compute() and autosave.scheduleX. Repository-composite operations that genuinely span multiple records remain on Repository (Contract 3) and are wrapped in orchestration.ts below; those wrappers are async because they go through Repository's Dexie transactions.
  2. attemptTransition refined: synchronous and takes a precomputed ComputeResult (for the frame_to_argument case) or no compute input (for the other three cases). Contract 4's attemptTransition(...): Promise<TransitionResult> is async, with an undocumented relationship to compute(). Two constitutional facts make the original signature unworkable. (a) The frame-to-argument validation gate (D3) consumes the C5 strict ValidationResult[] produced by compute() phase 1; if attemptTransition called compute() internally, it would either read the clock (violating Article II § 2 once the runtime requires computed_at) or duplicate the computed_at threading the state layer already does, breaking the single-clock-read property that the state spec made the realization of Article II § 2 in production code. (b) The architectural-mode-change and flavor-change scans operate purely on the FrameVersion's nodes and the (optional) ArgumentSession's premises; they are synchronous from the start. The constitutionally clean resolution is for attemptTransition to accept the precomputed compute output as a dependency, exactly as runFrameAction / runSessionAction accept it (and as computeInterviewOrder does, item 3 below). The signature becomes synchronous; the dependency graph in Contract 4 is unchanged (modes still imports @/runtime for the types ComputeResult and ValidationResult — modes does not call compute()). The state layer constructs the input by calling its compute_driver once and passing the result through to attemptTransition; for argument_to_frame, architectural, and flavor, the parameter is unused and the state layer passes undefined. This matches the I.5 refinement's reasoning: the modes layer does not perform side-effecting operations or read the clock; the state layer does.
  3. computeInterviewOrder refined: takes a precomputed ComputeResult. Contract 4's computeInterviewOrder(frame, session): InterviewItem[] requires the reachable set, the active set, and the status map to determine which nodes are open items and how to prioritize them; those are exactly the three projections in ComputeResult. Calling compute() internally would again either read the clock (forbidden) or duplicate computed_at threading. The same resolution as item 2 applies: the function takes ComputeResult as an explicit parameter and is pure. The state layer's SessionStoreSnapshot already carries compute_result per stream_i_state_spec_v1.html, so the call site is computeInterviewOrder(frame_version, session, compute_result) with no additional plumbing.

None of items 1–3 reverses a prior stream decision; each is a precision item under the same family as F-003 and the I.3 / I.4 / I.5 pending flags. The Article V § 3 trigger-recheck is in the self-review at the end of this document.

// src/modes/frame-actions.ts

The FrameActionDispatchTable implementation. Each entry is a synchronous pure function of ({ current_frame, current_version, ...patch_args, deps }) returning FrameTransformResult { next_frame, next_version, cascade? }. The state spec's action-runner.ts is the single caller: it destructures the patch, calls the matching entry, and threads the result through validateOnly(next_version, compute_driver) + AutosaveController.scheduleFrameSave(PendingFrameSave). No entry calls Repository, compute(), or any clock function directly; the table is pure and easily property-tested.

Every entry shares a four-step skeleton: (1) read deps.now() into a local at exactly once; (2) compute the post-mutation Node[] and Edge[] arrays (the entry's distinctive work); (3) build the next FrameVersion via nextFrameVersion(current_version, { nodes, edges, deps, at }); (4) build the next Frame by spreading current_frame with the new current_version_id and updated_at = at, plus any Frame-level field changes the entry makes (metadata_edited, default_policy_edited). Steps (1), (3), and (4) are identical across entries; step (2) is per-action.

/**
 * Typed error thrown by node_deleted when the deletion would cascade beyond
 * the node itself and its directly-incident edges, and the caller did not
 * pass cascade_confirmed: true. The error carries the cascade report so the
 * state layer can present the confirmation UI without re-running the cascade.
 */
export class CascadeConfirmationRequired extends Error {
  readonly kind = "cascade_confirmation_required" as const;
  constructor(readonly report: CascadeReport, readonly target_node_id: NodeRef) {
    super(`Deletion of node ${target_node_id} cascades to ` +
          `${report.deleted_node_ids.length} nodes and ` +
          `${report.deleted_edge_ids.length} edges; cascade_confirmed required.`);
  }
}

/**
 * Build the next FrameVersion record. Pure: parent_version_id is the current
 * version's id; version_number is incremented by one; is_milestone is false
 * (the autosave path); change_summary is undefined (the state layer fills it
 * in for milestone forces); created_at is the caller-supplied `at`. The new
 * id is freshly generated. The llm_settings_snapshot field, if present on the
 * current version, is forwarded verbatim — F-001 calibrated thresholds carry
 * forward unless an explicit G5 hook commit produces a new snapshot.
 */
export function nextFrameVersion(
  current: FrameVersion,
  args: { nodes: Node[]; edges: Edge[]; deps: FrameActionDeps; at: string },
): FrameVersion {
  return {
    id: args.deps.generateId() as FrameVersionId,
    frame_id: current.frame_id,
    version_number: current.version_number + 1,
    parent_version_id: current.id,
    created_at: args.at,
    is_milestone: false,
    nodes: args.nodes,
    edges: args.edges,
    ...(current.llm_settings_snapshot
      ? { llm_settings_snapshot: current.llm_settings_snapshot }
      : {}),
  };
}

/**
 * Replace one Node in an array by id, preserving the array's order. Returns a
 * new array; the original is not mutated. If `node_id` is not present, returns
 * the input array unchanged (no throw — calling this with a stale id is a
 * caller bug the action runner surfaces, not an invariant violation here).
 */
export function replaceNode(nodes: ReadonlyArray<Node>, node_id: NodeRef,
                            patch: Partial<Node>): Node[] {
  let found = false;
  const next: Node[] = [];
  for (const n of nodes) {
    if (n.id === node_id) {
      // Type-narrowing depends on the discriminant `type`; the patch must not
      // change `type`. The action runner enforces this; the helper trusts it.
      next.push({ ...(n as Node), ...(patch as Partial<Node>) } as Node);
      found = true;
    } else {
      next.push(n);
    }
  }
  return found ? next : (nodes as Node[]);
}

/**
 * Remove a set of node ids plus a set of edge ids from a FrameVersion's
 * collections in one pass. Stable: preserves the surviving ordering.
 */
export function removeNodeAndEdges(
  nodes: ReadonlyArray<Node>, edges: ReadonlyArray<Edge>,
  remove_node_ids: ReadonlySet<NodeRef>, remove_edge_ids: ReadonlySet<EdgeRef>,
): { nodes: Node[]; edges: Edge[] } {
  const out_nodes: Node[] = [];
  for (const n of nodes) if (!remove_node_ids.has(n.id)) out_nodes.push(n);
  const out_edges: Edge[] = [];
  for (const e of edges) {
    if (remove_edge_ids.has(e.id)) continue;
    // Edges incident to a removed node are also removed even if not explicitly
    // listed; this is the deletion contract.
    if (remove_node_ids.has(e.source) || remove_node_ids.has(e.target)) continue;
    out_edges.push(e);
  }
  return { nodes: out_nodes, edges: out_edges };
}

export const frameActions: FrameActionDispatchTable = {
  node_added({ current_frame, current_version, node, deps }) {
    const at = deps.now();
    // The node carries its own id (generated by the state layer's patch
    // construction site, which is the caller and the only place that needs
    // to know the new id ahead of time). The dispatch entry trusts the id
    // and appends.
    const next_nodes: Node[] = [...current_version.nodes, node];
    const next_version = nextFrameVersion(current_version, {
      nodes: next_nodes, edges: current_version.edges, deps, at,
    });
    const next_frame: Frame = {
      ...current_frame,
      current_version_id: next_version.id,
      updated_at: at,
    };
    return { next_frame, next_version };
  },

  node_edited({ current_frame, current_version, node_id, partial, deps }) {
    const at = deps.now();
    const next_nodes = replaceNode(current_version.nodes, node_id, partial);
    const next_version = nextFrameVersion(current_version, {
      nodes: next_nodes, edges: current_version.edges, deps, at,
    });
    const next_frame: Frame = {
      ...current_frame, current_version_id: next_version.id, updated_at: at,
    };
    return { next_frame, next_version };
  },

  node_deleted({ current_frame, current_version, node_id, cascade_confirmed, deps }) {
    const at = deps.now();
    const report = computeDeletionCascade(current_version, node_id);
    // "Trivial cascade" = the node itself plus its directly-incident edges
    // only, no transitive removals. Per Stream D D1, only SubQuestion and
    // Term cascade transitively (descendants reachable only through the
    // target). Everything else is trivial.
    const transitively_removed_nodes = report.deleted_node_ids.length - 1;  // minus the target itself
    const is_trivial = transitively_removed_nodes === 0;
    if (!is_trivial && !cascade_confirmed) {
      throw new CascadeConfirmationRequired(report, node_id);
    }
    const remove_node_ids = new Set(report.deleted_node_ids);
    const remove_edge_ids = new Set(report.deleted_edge_ids);
    const { nodes, edges } = removeNodeAndEdges(
      current_version.nodes, current_version.edges,
      remove_node_ids, remove_edge_ids,
    );
    const next_version = nextFrameVersion(current_version, { nodes, edges, deps, at });
    const next_frame: Frame = {
      ...current_frame, current_version_id: next_version.id, updated_at: at,
    };
    return {
      next_frame, next_version,
      cascade: {
        deleted_node_ids: [...sortedBy(report.deleted_node_ids, (id) => id)],
        deleted_edge_ids: [...sortedBy(report.deleted_edge_ids, (id) => id)],
      },
    };
  },

  edge_added({ current_frame, current_version, edge, deps }) {
    const at = deps.now();
    // V-EDGE-1 (connection-type validity) is enforced by the C5 validation
    // pass that the action runner invokes after every frame action; this
    // entry does not pre-screen, so the user's edit can surface as a
    // validation error rather than a thrown exception. This matches the
    // "editing strictness" rule in C5: edges are creatable freely; the
    // strictness-pre-transition gate is the only place that blocks.
    const next_edges: Edge[] = [...current_version.edges, edge];
    const next_version = nextFrameVersion(current_version, {
      nodes: current_version.nodes, edges: next_edges, deps, at,
    });
    const next_frame: Frame = {
      ...current_frame, current_version_id: next_version.id, updated_at: at,
    };
    return { next_frame, next_version };
  },

  edge_edited({ current_frame, current_version, edge_id, partial, deps }) {
    const at = deps.now();
    // Stream D D1 edge actions: source, target, and type are immutable
    // post-creation. The runner is responsible for filtering those keys out
    // of `partial` before calling; this entry does not re-enforce.
    const next_edges = current_version.edges.map((e) =>
      e.id === edge_id ? ({ ...(e as Edge), ...(partial as Partial<Edge>) } as Edge) : e
    );
    const next_version = nextFrameVersion(current_version, {
      nodes: current_version.nodes, edges: next_edges, deps, at,
    });
    const next_frame: Frame = {
      ...current_frame, current_version_id: next_version.id, updated_at: at,
    };
    return { next_frame, next_version };
  },

  edge_deleted({ current_frame, current_version, edge_id, deps }) {
    const at = deps.now();
    const next_edges = current_version.edges.filter((e) => e.id !== edge_id);
    const next_version = nextFrameVersion(current_version, {
      nodes: current_version.nodes, edges: next_edges, deps, at,
    });
    const next_frame: Frame = {
      ...current_frame, current_version_id: next_version.id, updated_at: at,
    };
    return { next_frame, next_version };
  },

  metadata_edited({ current_frame, current_version, partial, deps }) {
    const at = deps.now();
    // Frame metadata edits live on Frame; per Stream D D1 every persisted
    // change still creates a new FrameVersion (the autosave grain is the
    // version, even when the version contents are unchanged from the prior
    // version's nodes/edges). The new FrameVersion's `change_summary` is set
    // by the state-layer milestone path; here it remains undefined.
    const next_frame: Frame = { ...current_frame, ...partial, updated_at: at };
    const next_version = nextFrameVersion(current_version, {
      nodes: current_version.nodes, edges: current_version.edges, deps, at,
    });
    next_frame.current_version_id = next_version.id;
    return { next_frame, next_version };
  },

  default_policy_edited({ current_frame, current_version, node_type, policy, deps }) {
    const at = deps.now();
    const next_defaults = {
      ...(current_frame.default_satisfaction_policies ?? {}),
      [node_type]: policy,
    };
    const next_frame: Frame = {
      ...current_frame,
      default_satisfaction_policies: next_defaults,
      updated_at: at,
    };
    const next_version = nextFrameVersion(current_version, {
      nodes: current_version.nodes, edges: current_version.edges, deps, at,
    });
    next_frame.current_version_id = next_version.id;
    return { next_frame, next_version };
  },

  options_box_edited({ current_frame, current_version, node_id, policy, deps }) {
    const at = deps.now();
    // `policy: null` clears the per-instance override (the effective policy
    // then resolves to the frame's default for the node's type, per Stream A
    // A3's resolution order).
    const patch = policy === null
      ? ({ options_box: undefined } as Partial<Node>)
      : ({ options_box: policy } as Partial<Node>);
    const next_nodes = replaceNode(current_version.nodes, node_id, patch);
    const next_version = nextFrameVersion(current_version, {
      nodes: next_nodes, edges: current_version.edges, deps, at,
    });
    const next_frame: Frame = {
      ...current_frame, current_version_id: next_version.id, updated_at: at,
    };
    return { next_frame, next_version };
  },

  presentation_hint_edited({ current_frame, current_version, node_id, partial, deps }) {
    const at = deps.now();
    // Per Stream H H2 the presentation hints (x, y, collapsed) are versioned,
    // so this dispatch entry follows the same path as any other node edit —
    // a new FrameVersion is created. The state layer's debounce window
    // collapses rapid drags into a single FrameVersion in practice.
    const next_nodes = replaceNode(current_version.nodes, node_id, partial as Partial<Node>);
    const next_version = nextFrameVersion(current_version, {
      nodes: next_nodes, edges: current_version.edges, deps, at,
    });
    const next_frame: Frame = {
      ...current_frame, current_version_id: next_version.id, updated_at: at,
    };
    return { next_frame, next_version };
  },
};

Behavioral contracts on edge cases. Stale ids: node_edited / options_box_edited / presentation_hint_edited called with a node_id not in current_version.nodes return a next_version with the same nodes as before (the helper returned the input array unchanged) but with a new id and bumped version_number. The state layer's action-runner test asserts that a stale-id dispatch is a caller bug; in production the patch construction site (in frame-store.ts's applyEdit path) always reads from the snapshot before constructing the patch, so the id is fresh. Concurrent edits: the dispatch table is stateless, so concurrent calls produce two independent FrameTransformResults; the autosave controller's "latest payload wins" rule per stream_i_persistence_spec_v1.html determines which one persists. Identifier injection: all generated ids flow through deps.generateId() exactly. Clock reads: each dispatch entry reads deps.now() exactly once, into a local at, which is used as both next_version.created_at and next_frame.updated_at — preserving the single-clock-read property that defends Article II § 2 in production.

// src/modes/session-actions.ts

The SessionActionDispatchTable implementation. Each entry is a synchronous pure function of ({ current_session, current_version, ...patch_args, deps }) returning SessionTransformResult { next_session_raw, next_version_raw }. The "raw" suffix is significant: the entry produces an ArgumentSession with the user-data fields updated (premises, argument_edges, session_authorities, checkpoint_responses, interpretation_selections, metadata) but with the computed projections (status_map, output, active_path) left unset. The state-layer action runner reads deps.now() exactly once into at, stamps created_at = at on the version and updated_at = at on the session, invokes compute_driver.runFor(next_session_raw, at) to obtain the ComputeResult, writes the three projections back onto next_session_raw, and persists the assembled entities. This division enforces the rule that modes does not call compute().

Every entry shares a three-step skeleton: (1) compute the next argument-layer collection(s) (the entry's distinctive work); (2) build next_version_raw via nextSessionVersionRaw(current_version, { ...next_collections, deps }); (3) build next_session_raw by spreading current_session with the new current_version_id plus the argument-layer fields the action touched. The session-layer fields the schema persists alongside the version-projection — premises, argument_edges, session_authorities, checkpoint_responses, interpretation_selections — exist on ArgumentSession as the "current" projection of the latest version (per stream_b_schema_v1.html B4), so the entry updates them on the session and on the new version in lockstep.

import type {

/**
 * Build the next ArgumentSessionVersion's raw shell. The state-layer action
 * runner overwrites `created_at` immediately after this returns; the field is
 * set here to a sentinel that will be replaced. `is_milestone: false` is the
 * autosave-path default; the milestone path overwrites this too. The
 * `output_overrides` field (F-002) is intentionally not carried forward from
 * the prior version — G6's rewrite is invoked explicitly per output-time hook
 * activation, not preserved as inertia.
 */
export function nextSessionVersionRaw(
  current: ArgumentSessionVersion,
  args: {
    premises: ReadonlyArray<Premise>;
    argument_edges: ReadonlyArray<Edge>;
    session_authorities?: ReadonlyArray<Authority>;
    checkpoint_responses: ReadonlyArray<CheckpointResponse>;
    interpretation_selections: ReadonlyArray<InterpretationSelection>;
    deps: SessionActionDeps;
  },
): ArgumentSessionVersion {
  return {
    id: args.deps.generateId() as SessionVersionId,
    session_id: current.session_id,
    version_number: current.version_number + 1,
    parent_version_id: current.id,
    created_at: "",                         // overwritten by action runner
    is_milestone: false,
    premises: [...args.premises],
    argument_edges: [...args.argument_edges],
    session_authorities: args.session_authorities
      ? [...args.session_authorities] : undefined,
    checkpoint_responses: [...args.checkpoint_responses],
    interpretation_selections: [...args.interpretation_selections],
  };
}

/** Replace one Premise by id; preserve order. Pure. */
export function replacePremise(
  premises: ReadonlyArray<Premise>, premise_id: NodeRef,
  patch: Partial<Premise>,
): Premise[] {
  return premises.map((p) => p.id === premise_id ? { ...p, ...patch } : p);
}

/** Drop argument edges matching a predicate. Pure. */
export function removeArgumentEdgesByPredicate(
  edges: ReadonlyArray<Edge>, predicate: (e: Edge) => boolean,
): Edge[] {
  return edges.filter((e) => !predicate(e));
}

// src/modes/cascade.ts

The D1 cascade-summary computation. Stream D's per-node-type deletion rules divide into two regimes:

  • Trivial cascade. RootQuestion (cannot delete), Interpretation, Checkpoint, LogicalGate, Conclusion, Authority, Premise: deleting the node removes the node itself and all directly-incident edges; no further nodes are removed. Premise lives only in the argument layer and is not handled here.
  • Transitive cascade. SubQuestion and Term: deleting the node removes the node, its directly-incident edges, and every descendant reachable from the original root only through the deleted node. "Only through" is the reachability difference: nodes that were reachable from RootQuestion in the original frame but become unreachable if the target is removed.

The reachability difference is computed once per call. The algorithm: build the directed structural graph from the frame's nodes and edges, walk reachable-from-root in the original graph (R1), walk reachable-from-root in the graph with the target and its incident edges removed (R2), report R1 ∖ R2 as the cascade. This covers SubQuestion (its descendants), Term (its child Interpretations and anything those alone reached), and any chain of transitively-only-reached nodes. The two trivial-cascade-only types (Interpretation, Checkpoint, etc.) also use this algorithm; the result for them is exactly {target} by construction (since their downstream nodes are reachable from elsewhere or LEADS_TO targets are considered non-owning per Stream D D1 "the downstream nodes pointed at by its LEADS_TO edges are not deleted"). The algorithm is therefore uniform; the per-node-type rule difference is absorbed into the graph definition.

The structural graph used for the reachability walk: nodes are Node[]; directed edges include DECOMPOSES_INTO (Question → SubQuestion or Term), TURNS_ON (Question → Term), INTERPRETED_AS (Term → Interpretation), LEADS_TO (Interpretation/Question/Gate → Checkpoint/SubQuestion/Conclusion), GATES (Gate → target), and CheckpointOption.target_node_id as a virtual edge per Stream D D1 decision #3. Edges not included in the structural walk: FORECLOSES (a foreclosure edge does not establish reachability — it removes it), SUPPORTS, CONTRADICTS (argument-layer; not present in the frame's edges array — frame validation rejects them), ANSWERS (argument-layer), CITES, DISTINGUISHED_BY, BINDING_IN (Authority-related; not part of the question → conclusion path). LogicalGate inputs (AndGate.inputs, etc.) are walked as forward edges so that a SubQuestion contributing to a gate isn't kept alive by the gate but by its own structural parent; a node that exists only as a gate input loses its parent edge when the parent is deleted and falls out of R2.

import type { FrameVersion, Node, NodeRef, Edge, EdgeRef } from "@/schema";

export interface CascadeReport {
  /** All node ids that would be removed by the deletion, including the
      target. Sorted lexicographically. */
  deleted_node_ids: NodeRef[];
  /** All edge ids that would be removed: all edges incident to any deleted
      node, plus any explicit edges removed for type-specific reasons (e.g.,
      Interpretation's outgoing FORECLOSES edges are gone with the
      Interpretation). Sorted lexicographically. */
  deleted_edge_ids: EdgeRef[];
}

/**
 * Compute the deletion cascade for a target node within a FrameVersion.
 * Pure: no side effects, no clock reads, no LLM. Output order is determined
 * lexicographically so the report's serialization is stable.
 *
 * Algorithm:
 *   1. Locate the frame's RootQuestion.
 *   2. Build a forward-adjacency representation of the structural graph
 *      (excluding FORECLOSES and Authority-related edges per the spec
 *      above). Include virtual edges from Checkpoint to each option's
 *      target_node_id.
 *   3. Walk reachable-from-root once to produce R1 (original reachability).
 *   4. Remove the target and its incident edges from the adjacency.
 *   5. Walk again to produce R2 (post-deletion reachability).
 *   6. Cascade = R1 ∖ R2; always includes the target.
 *   7. Edge cascade = every edge in `frame.edges` whose source or target is
 *      in the cascade node set, plus every CheckpointOption-virtual-edge
 *      whose host Checkpoint or target is in the cascade.
 *      (Virtual edges are not in `frame.edges`; the report does not include
 *      them — the host Checkpoint's `options[].target_node_id` field will be
 *      preserved on the surviving Checkpoint if the host survives, or
 *      removed alongside the host if it does not.)
 */
export function computeDeletionCascade(
  frame: FrameVersion, target: NodeRef,
): CascadeReport;

Implementation contract for computeDeletionCascade:

  1. Root identification. Find the unique Node with type === "RootQuestion" (V-FR-1 guarantees exactly one). If absent (a malformed frame mid-edit), return { deleted_node_ids: [target], deleted_edge_ids: [edges incident to target sorted by id] } — the trivial-cascade fallback.
  2. Adjacency build. Iterate frame.edges sorted by edge id; for each, if its type is in the structural set (DECOMPOSES_INTO, TURNS_ON, INTERPRETED_AS, LEADS_TO, GATES), record a forward edge source → target. Iterate frame.nodes sorted by node id; for each LogicalGate, record forward edges from the gate to its named-slot inputs (AndGate.inputs[], NotGate.input, IfThenGate.antecedent & .consequent, UnlessGate.main & .exception); for each Checkpoint, record forward edges from the Checkpoint to each option's target_node_id (when set).
  3. Walk one. BFS or DFS from RootQuestion's id over the adjacency; collect reachable ids into a set R1.
  4. Removed adjacency. Construct a second adjacency that drops every edge whose source or target equals the deletion target; also drops every gate-input or checkpoint-option edge whose source or target equals the target.
  5. Walk two. Same BFS/DFS, this time over the removed adjacency. Collect R2.
  6. Cascade. cascade = R1 ∖ R2 ∪ {target} (the target is unconditionally included even if it was already unreachable in R1, which can happen mid-edit when a node is created but not yet wired up). Return deleted_node_ids as the sorted array of this set.
  7. Edge cascade. Iterate frame.edges sorted by edge id; collect every edge whose source or target is in the cascade set. Return as sorted array. Edge ids alone — the CheckpointOption.target_node_id fields are not edge ids and do not appear here; they are cleaned up on the surviving Checkpoint by the action runner's post-cascade pass if any of its target_node_ids pointed at a deleted node, by clearing those target_node_ids to undefined.

The post-cascade CheckpointOption.target_node_id cleanup is the action runner's responsibility, not frame-actions.ts directly: when node_deleted returns, the runner iterates the surviving next_version.nodes for Checkpoints whose option target_node_ids pointed into the cascade set, and clears those targets to undefined in a follow-up node_edited-equivalent transformation. This is mechanical and could equally well live inside node_deleted itself; it is hoisted to keep node_deleted's skeleton parallel to every other entry's. (Implementation choice — the I.6 coding session may inline it.)

Determinism note. The two reachability walks visit nodes in sorted order (the adjacency is built from sorted inputs, and the queue/stack uses the sorted order as well); the result is therefore byte-identical for a given (frame, target) input. Property test: cascade is monotonic — adding an edge that re-roots an orphan node into the reachable component shrinks the cascade (R2 grows). Property test: cascade is idempotent under repeated invocation with the same frame and target.

The four async wrappers around Repository's composite methods. The wrappers exist so that the state layer (and downstream consumers) sees a single import surface (@/modes) for both the per-action dispatch tables and the cross-entity composite operations, and so that orchestration-only argument shaping (most notably the orphan-candidate enumeration that migrateSession runs to populate the user-facing resolution UI before the user supplies the OrphanResolution[] array) lives next to its caller rather than crossing the modes / persistence boundary in the wrong direction. The wrappers contain no business logic of their own beyond argument shaping: every state mutation is performed inside the Repository method, which is in turn implemented inside a single Dexie transaction (stream_i_persistence_spec_v1.html "Atomicity contracts" § 1–4).

The wrappers are async because the underlying Repository methods are async. They are the only async exports in src/modes/. Every other modes export is a synchronous pure function; the state layer awaits these wrappers directly without going through the dispatch-table action runner, since they bypass the FramePatch / SessionPatch shape (they produce or fork entire entities at once, not field patches against a single working version).

Behavioral contracts for the orchestration wrappers (each tested in tests/modes/orchestration.test.ts):

  • Pass-through fidelity. Each of the four async wrappers passes its arguments to the underlying Repository method unmodified and returns the underlying method's result unmodified. The wrappers exist for symmetry of the modes-layer import surface, not to add semantics. Mock-Repository tests assert that the wrapper invokes the named method exactly once with exactly the supplied arguments and propagates the resolved value.
  • Error propagation. Each wrapper propagates QuotaExceededError and RepositoryError (the typed errors from @/persistence) without wrapping. The state layer catches at the call site and routes to the failure UI per stream_i_persistence_spec_v1.html "Quota and failure handling."
  • Orphan candidate determinism. enumerateOrphanCandidates(session, target_frame_version) is a synchronous pure function: two invocations against the same inputs produce byte-identical output (including the order of candidates in the returned array). Property test: shuffling the iteration order of the inputs (via differently-ordered session.current_version.checkpoint_responses[] arrays whose contents are the same) does not change the output, because the function sorts internally before walking.
  • Orphan-candidate / migrate round-trip. If enumerateOrphanCandidates(session, target) returns [], then migrateSession(repo, session.id, target.id, []) succeeds and the resulting ArgumentSessionVersion has no unresolved references (every checkpoint_response.checkpoint_id, interpretation_selection.term_id, and argument_edge.source/target appears in the new version's frame_version_snapshot.nodes). Conversely, if it returns n candidates and the caller supplies a complete OrphanResolution[] of length n, the resulting version has no unresolved references regardless of which resolution kinds were chosen.
  • Suggested-kind heuristic is advisory. The suggested_kind field on each OrphanCandidate is for UI pre-selection only; the persistence layer ignores it. The user-supplied OrphanResolution.kind is what determines disk state. The state layer is free to override the heuristic in the UI based on user preference settings (none currently defined; the heuristic is the only signal in v1).

D3 covers four distinct mode transitions: Frame Building → Argument Running (gated by the C5 strict pre-mode-transition validation check); Argument Running → Frame Building (always permitted, with a drift advisory if the underlying frame has newer versions); architectural mode change (legal ↔ general, gated by direction-mismatch resolution with inline editors); and flavor change (personal ↔ academic within general mode, always permitted with vocabulary advisories). attemptTransition dispatches on kind and returns a TransitionResult the state layer applies. The function is purely a gating-plus-advisory check: it does not itself commit a mode change to disk. The caller, on ok === true, performs the actual state mutation through the appropriate dispatch-table action (metadata_edited for an architectural-mode or flavor commit; the Argument Running entry-point UI for the operating-mode change, which is a UI-layer routing concern with no schema field to flip).

Signature refinement (in-session flag #2, F-003 family). Contract 4 declares attemptTransition as returning Promise<TransitionResult> and taking only the entities; this spec refines it to a synchronous function that takes a precomputed ComputeResult (for the frame_to_argument case, which reads validation_results) or omits the parameter when not needed (the other three cases produce their own gating signals from the entities alone). The refinement parallels I.5's discipline that the state layer is the single application caller of compute(): attemptTransition participates in the same single-clock-read-per-dispatch contract by accepting the freshest available ComputeResult rather than re-invoking compute() from inside the modes layer (which would either double-read the clock or require modes to bypass the state layer's compute-projection cache, both undesirable). The function remains pure synchronous; the resolutions? parameter from Contract 4's signature is reserved for v2 (no current case consumes orphan resolutions at the transition gate — orphan resolutions flow through migrateSession, which is the post-architectural-edit migration entrypoint, not a transition gate).

import type {
  FrameVersion, ArgumentSession, Mode, Flavor, NodeRef,
} from "@/schema";
import type { ComputeResult, ValidationResult } from "@/runtime";

export const TRANSITION_KINDS = [
  "frame_to_argument",
  "argument_to_frame",
  "architectural",
  "flavor",
] as const;
export type TransitionKind = (typeof TRANSITION_KINDS)[number];

export interface TransitionResult {
  ok: boolean;
  blocking: ValidationResult[];   // gating items; transition not committed
  advisory: ValidationResult[];   // shown to user, not gating
  /** Set when kind === "architectural"; per-Conclusion inline-editor seed
   *  payloads the UI uses to drive the resolution screen. Empty when no
   *  direction mismatches exist (the architectural change can commit silently
   *  if blocking[] is also empty). */
  inline_editors?: ConclusionDirectionEditor[];
}

export interface ConclusionDirectionEditor {
  node_id: NodeRef;
  current_direction_kind: "legal" | "general";
  required_direction_kind: "legal" | "general";
  /** When required_direction_kind === "legal": the legal enum values the user
   *  picks from (per Stream B B1 LegalDirection). When "general": the Position
   *  ids the user picks from (from frame.positions[]). */
  options: { value: string; label: string }[];
}

/**
 * The transition gating function. Synchronous and pure. The caller threads in
 * the freshest available ComputeResult when kind === "frame_to_argument";
 * other kinds may pass undefined.
 *
 * Returns a TransitionResult the caller acts on:
 *   - ok === true with empty blocking[]: the caller proceeds with the
 *     transition (the operating-mode change in the UI, or the dispatch-table
 *     action that commits the architectural / flavor change).
 *   - ok === false with non-empty blocking[]: the caller surfaces the
 *     blocking items in the appropriate modal (validation-errors list for
 *     frame_to_argument; inline-editors resolution screen for architectural).
 *     advisory[] is always shown alongside.
 *
 * The function performs no Repository access and no clock reads.
 */
export function attemptTransition(
  kind: TransitionKind,
  current: {
    frame: FrameVersion;
    session?: ArgumentSession;
    mode?: Mode;
    flavor?: Flavor;
  },
  target: { mode?: Mode; flavor?: Flavor },
  precomputed?: { compute_result?: ComputeResult; validation?: ValidationResult[] },
): TransitionResult {
  switch (kind) {
    case "frame_to_argument":
      return attemptFrameToArgument(current.frame, precomputed);
    case "argument_to_frame":
      return attemptArgumentToFrame(current.frame, current.session);
    case "architectural":
      return scanArchitecturalModeChange(current.frame, current.mode!, target.mode!);
    case "flavor":
      return scanFlavorChange(current.frame, current.flavor!, target.flavor!);
  }
}

/**
 * D3 "Frame Building → Argument Running." Gated by C5 strict pre-mode-
 * transition severity: any ValidationResult with severity "error" in the
 * precomputed ComputeResult.validation_results is blocking; severity
 * "warning" is advisory; severity "info" is dropped. The caller surfaces
 * blocking[] in a validation-errors modal with links to the implicated nodes
 * (the UI consumes ValidationResult.node_refs per Stream C ValidationResult).
 *
 * If precomputed.compute_result is absent, this function falls back to
 * precomputed.validation. If both are absent, throws — the caller MUST
 * supply at least one. This is deliberate: silently running with an empty
 * validation set would let an invalid frame slip into Argument Running,
 * which the C5 strict check exists to prevent.
 */
function attemptFrameToArgument(
  frame: FrameVersion,
  precomputed: { compute_result?: ComputeResult; validation?: ValidationResult[] } | undefined,
): TransitionResult {
  const validation =
    precomputed?.compute_result?.validation_results ??
    precomputed?.validation;
  if (!validation) {
    throw new Error(
      "attemptTransition('frame_to_argument'): precomputed.compute_result " +
      "or precomputed.validation must be supplied — modes does not call " +
      "compute() (see in-session flag #3, F-003 family).",
    );
  }
  const blocking = validation.filter((v) => v.severity === "error");
  const advisory = validation.filter((v) => v.severity === "warning");
  return { ok: blocking.length === 0, blocking, advisory };
}

/**
 * D3 "Argument Running → Frame Building." Always permitted (the user freely
 * navigates back to the frame editor). If the underlying frame has newer
 * versions than the session's frame_version_snapshot, an advisory is
 * surfaced ("This session is built against version N; the frame is now at
 * version N+1"); the user dismisses or initiates migration via the D2
 * action. No blocking items in this kind.
 */
function attemptArgumentToFrame(
  frame: FrameVersion,
  session: ArgumentSession | undefined,
): TransitionResult {
  // Drift detection requires comparing session.frame_version_id against the
  // frame's current_version_id. The caller passes in the current frame; if
  // there's drift, an advisory ValidationResult is constructed inline.
  const advisory: ValidationResult[] = [];
  if (session && session.frame_version_id !== frame.id) {
    advisory.push({
      rule_id: "MODE-DRIFT-ADVISORY",
      severity: "warning",
      message:
        \`This session is built against frame version \${session.frame_version_id}; \` +
        \`the frame is now at version \${frame.id}. Consider migrating before resuming.\`,
      node_refs: [],
    });
  }
  return { ok: true, blocking: [], advisory };
}

/**
 * D3 "Architectural mode change." Enumerates per the Stream D mode-change
 * scan: Conclusion direction-kind mismatches (blocking, with inline editors);
 * Premise kinds outside the new vocabulary (advisory, argument-layer); legal-
 * only Authority fields (advisory; absent ≡ null per B); legal-only flags
 * (Checkpoint.requires_authority, SubQuestion.is_jurisdictional, Question.
 * standard_of_review) when switching to general (advisory; inert in general
 * mode but data preserved per D3 "Data preservation").
 */
export function scanArchitecturalModeChange(
  frame: FrameVersion,
  current_mode: Mode,
  target_mode: Mode,
): TransitionResult {
  const blocking: ValidationResult[] = [];
  const advisory: ValidationResult[] = [];
  const inline_editors: ConclusionDirectionEditor[] = [];

  // Walk frame.nodes sorted by id for determinism.
  const nodes = [...frame.nodes].sort((a, b) => a.id.localeCompare(b.id));
  for (const n of nodes) {
    if (n.type === "Conclusion") {
      const dir_kind = n.direction.kind;
      const required_kind = target_mode === "legal" ? "legal" : "general";
      if (dir_kind !== required_kind) {
        blocking.push({
          rule_id: "MODE-CHANGE-CONCLUSION-DIRECTION",
          severity: "error",
          message:
            \`Conclusion '\${n.title ?? n.id}' has direction.kind = "\${dir_kind}"; \` +
            \`mode '\${target_mode}' requires "\${required_kind}".\`,
          node_refs: [n.id],
        });
        inline_editors.push({
          node_id: n.id,
          current_direction_kind: dir_kind,
          required_direction_kind: required_kind,
          options:
            required_kind === "legal"
              ? LEGAL_DIRECTION_VALUES.map((v) => ({ value: v, label: v }))
              : (frame.positions ?? []).map((p) => ({ value: p.id, label: p.label })),
        });
      }
    }
    // Premises live on the session, not the frame; advisory enumeration
    // for premise kinds happens at session-migration time, not here. The
    // Stream D mode-change scan lists premise items "for visibility" but
    // the architectural-mode change is a frame-level operation; sessions
    // pick up the advisory at their next migration recompute.
    if (n.type === "Checkpoint" && n.requires_authority === true && target_mode === "general") {
      advisory.push({
        rule_id: "MODE-CHANGE-REQUIRES-AUTHORITY-INERT",
        severity: "warning",
        message:
          \`Checkpoint '\${n.id}' has requires_authority = true; this flag is \` +
          \`legal-only and becomes inactive metadata in general mode (data preserved).\`,
        node_refs: [n.id],
      });
    }
    if (n.type === "SubQuestion" && n.is_jurisdictional === true && target_mode === "general") {
      advisory.push({
        rule_id: "MODE-CHANGE-IS-JURISDICTIONAL-INERT",
        severity: "warning",
        message:
          \`SubQuestion '\${n.id}' has is_jurisdictional = true; this flag is \` +
          \`legal-only and becomes inactive metadata in general mode (data preserved).\`,
        node_refs: [n.id],
      });
    }
    if (
      (n.type === "RootQuestion" || n.type === "SubQuestion") &&
      n.standard_of_review !== undefined &&
      target_mode === "general"
    ) {
      advisory.push({
        rule_id: "MODE-CHANGE-STANDARD-OF-REVIEW-INERT",
        severity: "warning",
        message:
          \`\${n.type} '\${n.id}' has standard_of_review set; this field is \` +
          \`legal-only and becomes inactive metadata in general mode (data preserved).\`,
        node_refs: [n.id],
      });
    }
    if (n.type === "Authority") {
      const has_legal_only =
        n.court !== undefined ||
        n.year !== undefined ||
        n.is_binding !== undefined ||
        n.binding_in !== undefined;
      if (has_legal_only && target_mode === "general") {
        advisory.push({
          rule_id: "MODE-CHANGE-AUTHORITY-LEGAL-FIELDS-INERT",
          severity: "warning",
          message:
            \`Authority '\${n.id}' has legal-only fields set (court, year, \` +
            \`is_binding, binding_in); these become inactive metadata in general mode.\`,
          node_refs: [n.id],
        });
      }
    }
  }

  return {
    ok: blocking.length === 0,
    blocking,
    advisory,
    inline_editors: inline_editors.length > 0 ? inline_editors : undefined,
  };
}

/**
 * D3 "Flavor change." Lighter than architectural-mode change. Schema is
 * unchanged; only Premise kind vocabulary, Authority visibility, and UI
 * strings shift. Always commits; surfaces advisories for Premises (session-
 * layer; advisory only at frame transition time) and for Authority field
 * visibility changes (frame-level; advisory only).
 *
 * Note: this scan does not look at sessions — flavor change is a frame-level

 * advisory at its next migration. The session-time check is V-ARG-3 in
 * Stream C.

export function scanFlavorChange(
  frame: FrameVersion,
  current_flavor: Flavor,
  target_flavor: Flavor,

    const authority_nodes = nodes.filter((n) => n.type === "Authority");
    if (authority_nodes.length > 0) {
      if (target_flavor === "personal") {
        advisory.push({
          rule_id: "FLAVOR-CHANGE-AUTHORITY-HIDDEN",
          severity: "warning",
          message:
            \`\${authority_nodes.length} Authority node(s) will be hidden by default \` +
            \`in personal flavor. Re-enable via the per-frame "enable Source fields" toggle.\`,
          node_refs: authority_nodes.map((n) => n.id),
        });
      } else if (target_flavor === "academic") {
        advisory.push({
          rule_id: "FLAVOR-CHANGE-AUTHORITY-RELABELED",
          severity: "warning",
          message:
            \`\${authority_nodes.length} Authority node(s) will be relabeled to "Source" \` +
            \`with academic-flavor field visibility (author, venue, position).\`,
          node_refs: authority_nodes.map((n) => n.id),
        });
      }
    }
  }

  return { ok: true, blocking: [], advisory };
}

Behavioral contracts for attemptTransition and its scanners:

  • Pure synchronous, deterministic. Every code path is a deterministic walk over sorted node arrays; given the same inputs, two invocations produce byte-identical TransitionResult objects (including the order of items in blocking[], advisory[], and inline_editors[]). No clock reads, no Math.random, no Repository access.
  • Single sort discipline. Every walk over frame.nodes sorts by id once at the top of the function; downstream filters preserve order. This satisfies the I.6 widening of the no-unsorted-iteration ESLint rule recommendation in the I.1 self-review.
  • Frame-to-argument refuses to silently succeed. If neither precomputed.compute_result nor precomputed.validation is supplied, the function throws. The caller (state layer) always has a fresh ComputeResult in its projection (per I.5); the throw exists to catch wiring bugs early in dev, not to surface in production.
  • Architectural-mode advisory completeness. The scan covers every Stream D D3 mode-change-scan item: direction mismatch (blocking with inline editor); requires_authority (advisory, legal→general); is_jurisdictional (advisory, legal→general); standard_of_review (advisory, legal→general); Authority legal-only fields (advisory, legal→general).   • Flavor change always commits. No code path in scanFlavorChange produces a blocking item. Premise vocabulary changes are session-layer concerns surfaced via V-ARG-3 at session time; this scan only looks at the frame.
  • Reserved resolutions? parameter. Contract 4's signature includes an optional resolutions?: OrphanResolution[] parameter. No v1 transition kind consumes this; it is reserved for a hypothetical future case where the transition gate itself applies resolutions inline (rather than the current pattern, where resolutions flow through migrateSession separately). The state-layer caller passes undefined for this parameter on every current call site; the modes layer ignores it. Logged in the in-session flag as part of the F-003 family (parameter is not removed from Contract 4; it is documented as reserved).

D2's computeInterviewOrder. Pure synchronous function of (frame, session, compute_result) that returns an ordered InterviewItem[] the UI consumes as the user-facing "what to address next" list. The function is purely derivational: it stores nothing, mutates nothing, and reads no clock. Every invocation against the same inputs returns a byte-identical array, including the recommended_next flag on the first non-advisory item.

Signature refinement (in-session flag #3, F-003 family). Contract 4 declares computeInterviewOrder(frame, session): InterviewItem[]. This spec refines it to computeInterviewOrder(frame, session, compute_result): InterviewItem[] for the same reason attemptTransition was refined: the function needs compute_result.reachable_set, compute_result.active_set, and compute_result.status_map per Stream C Phase 3 / Phase 4 to filter the DFS walk and to compute the reason field on each InterviewItem. Re-invoking compute() inside the interview function would double-read the clock per dispatch (the state layer already holds the freshest ComputeResult in its projection per I.5); the constitutionally clean resolution is for the state layer to thread its cached ComputeResult in as the third parameter. import type {
  FrameVersion, ArgumentSession, NodeRef, Node, NodeType,
} from "@/schema";
import type { ComputeResult } from "@/runtime";

export interface InterviewItem {
  node_id: NodeRef;
  is_jurisdictional: boolean;
  reason: "open" | "indeterminate" | "contested" | "best_inference_pending";
  recommended_next: boolean;
  /** Internal sort keys, exposed for the state layer's UI grouping (the UI
   *  presents jurisdictional items in a distinct top group; this field lets
   *  the UI partition without re-running the algorithm). */
  dfs_order: number;
  term_order: number;
}

export const INTERVIEW_SORT_KEYS = [
  "is_jurisdictional",
  "dfs_order",
  "term_order",
  "node_id",
] as const;

/**
 * D2 interview-ordering algorithm. Returns the open-items list sorted by:
 *   (1) is_jurisdictional descending (true before false) — the A8 priority bubble,
 *   (2) dfs_order ascending — pre-order from RootQuestion through the active path,
 *   (3) term_order ascending — for Term children of a Question with no selection,
 *       Term.order is the per-Question sort key (Stream D D2 "DFS branching"),
 *   (4) node_id lex ascending — final tiebreaker.
 *
 * The first item in the returned array has recommended_next === true; all
 * others have recommended_next === false. The recommendation is a UI hint;
 * the user is free to address any item.
 */
export function computeInterviewOrder(
  frame: FrameVersion,
  session: ArgumentSession,
  compute_result: ComputeResult,
): InterviewItem[] {
  const reachable = compute_result.reachable_set;       // Set<NodeRef>
  const status_map = compute_result.status_map;         // Map<NodeRef, NodeStatus>

  // Build per-node lookup for O(1) access during the DFS.
  const nodes_by_id = new Map(frame.nodes.map((n) => [n.id, n] as const));

  // DFS pre-order walk from RootQuestion. The walk visits each node at
  // most once (a Set guards against gate-input cycles, which shouldn't
  // exist in a valid frame but defending against them is cheap).
  const dfs_order_by_id = new Map<NodeRef, number>();
  const visited = new Set<NodeRef>();
  let cursor = 0;
  const root = frame.nodes.find((n) => n.type === "RootQuestion");
  if (root) {
    dfsPreorderFromRoot(root.id, nodes_by_id, session, visited, (id) => {
      dfs_order_by_id.set(id, cursor++);
    });
  }

  // Emit open items in DFS order, filtering by reachability and the open-
  // item predicate. term_order is read from the parent Question's child-
  // Term position (only meaningful for Term items; 0 for non-Term items).
  const items: InterviewItem[] = [];
  for (const [id, dfs_order] of dfs_order_by_id) {
    if (!reachable.has(id)) continue;
    const node = nodes_by_id.get(id)!;
    const item_seed = openItemFor(node, session, status_map);
    if (item_seed === null) continue;
    items.push({
      node_id: id,
      is_jurisdictional: isJurisdictionalAt(node, nodes_by_id),
      reason: item_seed.reason,
      recommended_next: false,
      dfs_order,
      term_order: termOrderOf(node, nodes_by_id),
    });
  }

  // Final four-key sort. JavaScript's Array.sort is stable since ES2019;
  // the keys above guarantee total order regardless.
  items.sort(
    (a, b) =>
      Number(b.is_jurisdictional) - Number(a.is_jurisdictional) ||
      a.dfs_order - b.dfs_order ||
      a.term_order - b.term_order ||
      a.node_id.localeCompare(b.node_id),
  );

  // Set recommended_next on the first non-advisory item. An "advisory"
  // item is one whose reason is "best_inference_pending" (per Stream D D2,
  // these are surfaced with lower priority). If every item is advisory,
  // the first item in the sorted list gets the recommendation.
  const first_non_advisory = items.findIndex((it) => it.reason !== "best_inference_pending");
  if (first_non_advisory >= 0) items[first_non_advisory].recommended_next = true;
  else if (items.length > 0) items[0].recommended_next = true;

  return items;
}

/**
 * Pre-order DFS from a starting node, with per-node-type branching rules
 * per Stream D D2 "DFS branching":
 *   - SubQuestion children: sorted by (is_jurisdictional desc, id lex);
 *   - Term children of a Question: sorted by (order asc, id lex);
 *   - Interpretation children of a Term: only descend into SELECTED ones;
 *     if no selection, do not descend (the Term itself is the open item);
 *   - CheckpointOption targets: only descend into the CHOSEN option's target;
 *     if no response, do not descend (the Checkpoint itself is the open item);
 *   - LogicalGate inputs: descended in named-slot order
 *     (AND/OR: list order; NOT: single input; IF_THEN: antecedent before
 *     consequent; UNLESS: main before exception).
 *
 * The visit function is called in pre-order (parent visited before children).
 */
export function dfsPreorderFromRoot(
  start_id: NodeRef,
  nodes_by_id: Map<NodeRef, Node>,
  session: ArgumentSession,
  visited: Set<NodeRef>,
  visit: (id: NodeRef) => void,
): void {
  if (visited.has(start_id)) return;
  visited.add(start_id);
  visit(start_id);

  const node = nodes_by_id.get(start_id);
  if (!node) return;

  const child_ids = orderedChildIdsOf(node, nodes_by_id, session);
  for (const child_id of child_ids) {
    dfsPreorderFromRoot(child_id, nodes_by_id, session, visited, visit);
  }
}

Behavioral contracts for computeInterviewOrder:

  • Determinism. The function is a synchronous pure walk over sorted node collections; given the same (frame, session, compute_result) inputs, two invocations produce byte-identical output. Property test: shuffling the iteration order of frame.nodes (with the same set of nodes) yields the same result because the DFS-from-RootQuestion walk does not depend on the input array's order — it uses the per-node-type branching rules above. (The implementation can additionally sort frame.nodes once at the top for defense in depth; the I.6 coding session may choose to inline this.)
  • Jurisdictional priority bubble (A8). Every reachable open item descended-from a SubQuestion with is_jurisdictional === true sorts before every non-jurisdictional item, regardless of DFS pre-order. Test: a frame where a jurisdictional SubQuestion is the third child of RootQuestion produces an open-items list whose first n items are the jurisdictional subtree's open items.
  • Single-Term branching uses Term.order. When a Question (Root or Sub) has multiple Term children and the user has not yet selected an Interpretation on any of them, the Terms are emitted in Term.order ascending sequence (Stream B B1.6 schema field), with lexicographic id as the tiebreaker. The InterviewItem.term_order field exposes this so the UI can render Term children of one Question as a numbered list without re-deriving the sort.
  • Selection-gated descent. The DFS descends past a Term only into the selected Interpretation(s) (per Stream D D2 "Interpretation children of a Term: only the selected"); it descends past a Checkpoint only into the chosen option's target_node_id (per "CheckpointOption targets: only the chosen"). Test: changing a CheckpointResponse from option A (whose target is Subtree X) to option B (whose target is Subtree Y) re-derives the interview list with Subtree Y's open items replacing Subtree X's.
  • Linked Terms are skipped. Per Stream D D2, a Term with linked_to set is skipped if its target Term is reachable; the target's selection covers it. The openItemFor function returns null for the linked Term when its target is in reachable_set and has a non-empty InterpretationSelection.
  • Advisory items sort last but appear. best_inference_pending items (Interpretations lacking authority/premise support per the options_box rule) appear in the list but with recommended_next: false; the recommended_next flag goes to the first non-advisory item. The UI surfaces advisories in a distinct visual treatment.
  • Empty list ⇒ session complete. Per Stream D D2 "Completion," an empty InterviewItem[] means the session is complete. No special "completion" flag is added to the return type; the empty array is sufficient signal.
  • Reachability respect. The function never emits an item for a node not in compute_result.reachable_set. Foreclosed subtrees (per Term dispositive resolution or LogicalGate evaluation) are filtered out; their open items disappear from the interview when foreclosed and reappear if the foreclosure is reversed (re-deriving the list against the new compute_result).

Self-review (Article VIII § 4)

Architectural commitments (Constitution Article II § 1–6)

  • § 1 Frame / argument split. The two dispatch tables are physically separated: frame-actions.ts never reads from or writes to argument-layer collections (premises, argument_edges, checkpoint_responses, interpretation_selections, session_authorities); session-actions.ts never writes to frame_version_snapshot (it reads it as a frozen frame projection). The frame-projection invariant in the session-actions.test.ts plan pins this. ✓
  • § 2 Determinism. No modes export reads the clock. frame-actions and session-actions derive new ids via deps.generateId(); tests inject a deterministic generator. cascade.ts, transitions.ts, and interview.ts are pure synchronous walks over sorted entity collections. The orchestration wrappers do not read the clock either — they delegate to Repository methods that read the clock at the persistence boundary (per the persistence spec self-review § 2). The three F-003-family refinements (signature changes that route ComputeResult through call sites rather than have modes call compute()) exist to preserve the single-clock-read-per-dispatch contract I.5 established. ✓
  • § 3 Per-node satisfaction criteria. options_box_edited (frame-action) and default_policy_edited (frame-action) handle the per-node and per-NodeType configurable options-box / satisfaction-policy edits respectively. The computeInterviewOrder open-item predicate consults the relevant options_box on Interpretation and Checkpoint nodes (via compute_result.status_map); the "best_inference_pending" reason fires when an Interpretation's options_box requires authority_required and no support exists. ✓
  • § 4 Independent versioning. Every frame-action returns a new FrameVersion (via nextFrameVersion helper); every session-action returns a new ArgumentSessionVersion (via nextSessionVersionRaw helper). The action runner in state composes these with persistence's saveFrameVersion / saveSessionVersion auto-save path. Prior versions are never mutated; this spec asserts the contract throughout and the per-dispatch-entry tests verify it. ✓
  • § 5 Human-readable structure. File map is seven small focused files. Each dispatch-entry is a top-level named export with documentation. The transition function dispatches on a four-element string enum visible at the top of the file. The interview algorithm reads as a literal translation of Stream D D2's "Ordering algorithm" pseudocode. ✓
  • § 6 Use-case commitment. The architectural-mode-change scan blocks on legal-direction mismatches because the legal use case requires that Conclusion.direction be one of a fixed legal enum; the general-mode commitment is symmetric. Personal / academic flavor changes are non-blocking because the legal use case does not constrain them. The interview-ordering algorithm's jurisdictional priority bubble (A8) is a legal-mode-specific surface; in general mode (where no SubQuestion has is_jurisdictional: true), the bubble silently does nothing. ✓
