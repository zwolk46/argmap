stream_i_schema_spec_v1.html

Per-module spec for src/schema/  ·  Stream I session I.2  ·  May 10, 2026

This document is the implementation source of truth for src/schema/. It commits per-file TypeScript types, the discriminated-union encodings for Node and Edge, the SatisfactionPolicy and Condition shapes, the full validation-rule registry, the JSON envelope, the migration registry, and the Vitest acceptance tests for the module. The cross-module surface (what other modules import from @/schema) is pinned in stream_i_contracts_v1.html Contract 1; this spec adds the per-file organization, internal types, and behavioral contracts the coding session implements against. The schema module is the dependency root: every other module imports from it, and it imports nothing from the application codebase.

How to read this

The coding session implementing src/schema/ reads, in order: Constitution and Protocol, current_state.html, stream_i_buildkit_index_v1.html, stream_i_contracts_v1.html Contract 1, this document. No prior stream documents (A–H) are read. Implement-until-green against the Vitest tests at the end of this document; write src/schema/README.md as the cross-session compression artifact for downstream sessions. This spec uses real TypeScript exclusively; all ? markers are TypeScript optional, and discriminated-union shapes use a type or kind field as the discriminator (each interface explicitly states which).

// src/schema/nodes.ts

The Node discriminated union over nine variants. The discriminator field is type. Within type: "LogicalGate", the sub-discriminator is gate_type (V-GATE-1..4 enforce arity per gate_type). Per the contradiction-fix already adopted in current_state.html, FORECLOSES is an edge type only, never a gate_type. The presentation nested object holds optional UI hints ignored by the runtime (x, y, collapsed, color_override); the layout module reads node.presentation?.collapsed and the user-anchored x/y from this nested object — it does not access flat fields on Node.

import type { NodeRef } from "./identifiers";
import type { Jurisdiction } from "./frame";
import type { SatisfactionPolicy } from "./satisfaction-policy";

export type NodeType =
  | "RootQuestion" | "SubQuestion" | "Term" | "Interpretation"
  | "Checkpoint" | "LogicalGate" | "Conclusion" | "Authority" | "Premise";

  presentation?: {
    x?: number;
    y?: number;
    color_override?: string;
    collapsed?: boolean;
  };
  created_at: string;                          // ISO-8601
  updated_at: string;
  created_by?: string;                         // user id; multi-user is v2
}

Checkpoint and CheckpointOption (B1.5)

export type CheckpointAnswerType = "boolean" | "multiple_choice" | "graded";

export interface CheckpointOption {
  id: string;                                  // stable within parent Checkpoint
  label: string;                               // "yes" | "no" | "unclear" | author-defined
  target_node_id?: NodeRef;                    // null only for the "graded" middle state
  satisfies: boolean;                          // whether selecting this option counts as satisfying the Checkpoint
  routes_to_status?: "contested";              // override for "graded" middle state
}

export type BurdenLevel =
  | "preponderance" | "clear_and_convincing" | "beyond_reasonable_doubt"
  | "scintilla" | "substantial_evidence";

export interface Checkpoint extends NodeBase {
  type: "Checkpoint";
  layer: "frame";
  question: string;
  answer_type: CheckpointAnswerType;
  options: CheckpointOption[];                 // shape and length depends on answer_type (V-NODE-5/6/7)
  requires_premise: boolean;                   // default true
  requires_authority: boolean;                 // default false; legal mode only (V-ARG-2)
  burden_level?: BurdenLevel;                  // legal mode; feeds burden_met condition
  options_box?: SatisfactionPolicy;
}

Defaults that Frame-Building actions seed: a new Checkpoint is created with answer_type: "boolean" and options [{ id, label: "yes", satisfies: true }, { id, label: "no", satisfies: false }] (per A2). Default for new multiple_choice Checkpoints is two options "yes"/"no" with satisfies mirroring the boolean default. graded Checkpoints have exactly three options; the middle option carries routes_to_status: "contested". Authors may relabel; the three-state structure is fixed.

LogicalGate (B1.6) — sub-discriminated union with named slots

Per the contradiction fix and V-GATE-1..4: gate inputs are formally typed via named slot fields specific to each gate_type. There is no uniform inputs[] on LogicalGate; the encoding below makes IF_THEN's antecedent/consequent and UNLESS's main/exception structurally explicit. The five gate variants share type: "LogicalGate" and discriminate on gate_type. Boolean-evaluable input nodes are: Checkpoint, Interpretation, SubQuestion, RootQuestion, and other LogicalGates. Term and Premise/Authority inputs are forbidden (V-GATE-5, V-GATE-6). LogicalGate has no per-instance options_box override; its satisfaction is computed from input statuses per gate_type semantics in Stream C.

export interface AndGate extends NodeBase {
  type: "LogicalGate";
  layer: "frame";
  gate_type: "AND";
  inputs: NodeRef[];                           // V-GATE-1: minimum 2
  output_target?: NodeRef;
}

export interface OrGate extends NodeBase {
  type: "LogicalGate";
  layer: "frame";
  gate_type: "OR";
  inputs: NodeRef[];                           // V-GATE-1: minimum 2
  output_target?: NodeRef;
}

export interface NotGate extends NodeBase {
  type: "LogicalGate";
  layer: "frame";
  gate_type: "NOT";
  input: NodeRef;                              // V-GATE-2: exactly 1
  output_target?: NodeRef;
}

export interface IfThenGate extends NodeBase {
  type: "LogicalGate";
  layer: "frame";
  gate_type: "IF_THEN";
  antecedent: NodeRef;                         // V-GATE-3: exactly one antecedent
  consequent: NodeRef;                         // V-GATE-3: exactly one consequent
  output_target?: NodeRef;
}

export interface UnlessGate extends NodeBase {
  type: "LogicalGate";
  layer: "frame";
  gate_type: "UNLESS";
  main: NodeRef;                               // V-GATE-4: exactly one main
  exception: NodeRef;                          // V-GATE-4: exactly one exception
  output_target?: NodeRef;
}

export type LogicalGate = AndGate | OrGate | NotGate | IfThenGate | UnlessGate;

Conclusion and ConclusionDirection (B1.7, A1)

Confidence is not a stored field on Conclusion (per A1 and current_state.html). It is computed at session render time from the status map (see ConfidenceBreakdown in session.ts) and displayed alongside the Conclusion. Storing it on the node would let users assert certainty the structure does not support — the explicit reasoning in A1 against this. The legal-mode direction enum and the general-mode position_id reference into Frame.positions are both required and discriminate on kind; mode mismatches at the Conclusion layer block architectural mode transitions per Stream D D3.

export type ConclusionDirection =
  | { kind: "legal";
      value: "favors_plaintiff" | "favors_defendant"
           | "affirm" | "reverse" | "remand" | "dismiss" | "custom";
      custom_label?: string;                   // required when value === "custom"
    }
  | { kind: "general";
      position_id: string;                     // references Frame.positions[].id (V-FR-10)
      custom_label?: string;
    };

export interface Conclusion extends NodeBase {
  type: "Conclusion";
  layer: "frame";
  statement: string;
  direction: ConclusionDirection;              // required; mode-aware
  reasoning_summary?: string;                  // LLM hook G3 commits here at build time
  tags?: string[];
  remedy?: string;                             // legal mode v1; promotable to Remedy node in v2
  options_box?: SatisfactionPolicy;
}

Authority (B1.8)

The same Authority shape is used whether the node lives on the Frame (cited at build-time supporting an Interpretation) or on the ArgumentSession (added at runtime by a Premise's authority_ref). The locus is implicit from where the node is stored. In personal flavor, the type is labeled "Source" in the UI; the schema name remains Authority for code consistency. BINDING_IN is implemented as the embedded binding_in: Jurisdiction[] field rather than as an edges-table row — the EdgeBase requires a NodeRef target, but Jurisdiction is a value object, not an entity (current_state.html / Stream B B2).

export interface Authority extends NodeBase {
  type: "Authority";
  layer: "frame" | "argument";                 // determined by where it is stored
  citation: string;                            // e.g., "Erie R.R. v. Tompkins, 304 U.S. 64 (1938)"

  // Legal-mode fields:
  court?: string;
  year?: number;
  holding_summary?: string;
  is_holding?: boolean;                        // vs dicta; default true
  is_binding?: boolean;                        // computed by F6 when live; manually settable v1
  jurisdiction?: Jurisdiction;                 // where this authority issues from (V-FR-11 when is_binding)
  binding_in?: Jurisdiction[];                 // embedded relation: see "BINDING_IN edge" note in B2

  // Academic-flavor fields:
  author?: string;
  venue?: string;                              // journal, conference, publisher
  position?: string;                           // what the author argues, or "represents scholarly consensus"

  // Universal:
  url?: string;
  short_label?: string;                        // for compact display, e.g. "Erie"
}

Premise (B1.9, A11)

PremiseKind is a runtime-checked string drawn from a vocabulary that depends on the parent frame's mode and flavor. The schema layer types kind as the open string union of all vocabularies; per-frame validity is enforced at validation time by V-ARG-3 against PREMISE_KIND_VOCABULARIES. This keeps the schema mode-agnostic at the type level and centralizes vocabulary in frame.ts.

export type PremiseKind =
  // legal:
  | "stipulated" | "found" | "disputed" | "procedural"
  // academic:
  | "empirical" | "definitional" | "normative"
  // personal:
  | "observation" | "value" | "assumption";
// (note: "stipulated" is shared across legal/academic/personal vocabularies)

export interface Premise extends NodeBase {
  type: "Premise";
  layer: "argument";
  statement: string;
  kind: PremiseKind;                           // V-ARG-3 enforces vocabulary by frame.mode/flavor
  source?: string;                             // free-text citation/source label
  authority_ref?: NodeRef;                     // id of an Authority node (frame-layer or session-layer); V-ARG-2/V-ARG-4
}

Node union and type guards

export type Node =
  | RootQuestion | SubQuestion | Term | Interpretation
  | Checkpoint | LogicalGate | Conclusion | Authority | Premise;

// Type guards. The discriminator is `type`; for LogicalGate, also narrow on gate_type.
export function isRootQuestion(n: Node): n is RootQuestion { return n.type === "RootQuestion"; }
export function isSubQuestion(n: Node): n is SubQuestion   { return n.type === "SubQuestion"; }
export function isTerm(n: Node): n is Term                 { return n.type === "Term"; }
export function isInterpretation(n: Node): n is Interpretation { return n.type === "Interpretation"; }
export function isCheckpoint(n: Node): n is Checkpoint     { return n.type === "Checkpoint"; }
export function isLogicalGate(n: Node): n is LogicalGate   { return n.type === "LogicalGate"; }
export function isConclusion(n: Node): n is Conclusion     { return n.type === "Conclusion"; }
export function isAuthority(n: Node): n is Authority       { return n.type === "Authority"; }
export function isPremise(n: Node): n is Premise           { return n.type === "Premise"; }

export function isAndGate(g: LogicalGate): g is AndGate         { return g.gate_type === "AND"; }
export function isOrGate(g: LogicalGate): g is OrGate           { return g.gate_type === "OR"; }
export function isNotGate(g: LogicalGate): g is NotGate         { return g.gate_type === "NOT"; }
export function isIfThenGate(g: LogicalGate): g is IfThenGate   { return g.gate_type === "IF_THEN"; }
export function isUnlessGate(g: LogicalGate): g is UnlessGate   { return g.gate_type === "UNLESS"; }

// Boolean-evaluable check used by V-GATE-5/6 and the runtime gate evaluator.
export function isBooleanEvaluable(n: Node): boolean {
  return n.type === "Checkpoint" || n.type === "Interpretation"
      || n.type === "SubQuestion" || n.type === "RootQuestion"
      || n.type === "LogicalGate";
}
// src/schema/edges.ts

Twelve edge types form the discriminated union Edge. The discriminator is type. Edges are always directed; bidirectionality is modeled as two separate Edge entities. Per current_state.html / Stream B B2: CITED_BY was renamed CITES, ALTERNATIVE_PATH_TO was removed (two convergent paths are two LEADS_TO edges with the same target), and FORECLOSES is an edge only (not a LogicalGate type). The BINDING_IN edge type is retained in the EdgeType union for the validation layer's enumeration but is implemented as the embedded Authority.binding_in: Jurisdiction[] field rather than as an Edge row — Jurisdiction is a value object, not an entity, and EdgeBase requires a NodeRef target.

import type { NodeRef, EdgeRef } from "./identifiers";

export type EdgeType =

  | "FORECLOSES" | "GATES" | "ANSWERS" | "SUPPORTS" | "CONTRADICTS"
  | "CITES" | "BINDING_IN" | "DISTINGUISHED_BY";

export interface EdgeBase {
  id: string;                                  // UUID v4
  slug?: string;
  type: EdgeType;
  layer: "frame" | "argument";
  source: NodeRef;
  target: NodeRef;
  label?: string;
  notes?: string;
  created_at: string;
  updated_at: string;
}

Valid source/target type pairs (V-EDGE-1 source data)

The validation rule V-EDGE-1 enforces that every edge's source and target node types are permitted for that edge type. The truth table below is the single source of consultation; the validation rule walks VALID_EDGE_PAIRS rather than encoding the rules per edge type. The drag-from-handle UI gesture (Stream E E2) consults the same map to filter the edge-type picker by source node type.

import type { NodeType } from "./nodes";

export interface ValidEdgePair {
  source_types: ReadonlyArray<NodeType>;
  target_types: ReadonlyArray<NodeType>;
}

export const VALID_EDGE_PAIRS: Readonly<Record<EdgeType, ValidEdgePair>> = {
  DECOMPOSES_INTO: { source_types: ["RootQuestion", "SubQuestion"],
                     target_types: ["SubQuestion"] },
  TURNS_ON:        { source_types: ["RootQuestion", "SubQuestion"],
                     target_types: ["Term"] },
  INTERPRETED_AS:  { source_types: ["Term"],
                     target_types: ["Interpretation"] },
  LEADS_TO:        { source_types: ["Interpretation", "LogicalGate"],
                     target_types: ["Checkpoint", "SubQuestion", "Conclusion", "LogicalGate"] },
  FORECLOSES:      { source_types: ["Interpretation"],
                     target_types: ["Term", "SubQuestion", "Checkpoint"] },
  GATES:           { source_types: ["LogicalGate"],
                     target_types: ["RootQuestion", "SubQuestion", "Term", "Interpretation",
                                    "Checkpoint", "LogicalGate", "Conclusion"] },
  ANSWERS:         { source_types: ["Premise"],
                     target_types: ["Checkpoint"] },
  SUPPORTS:        { source_types: ["Premise"],
                     target_types: ["Interpretation", "Conclusion"] },
  CONTRADICTS:     { source_types: ["Premise"],

  CITES:           { source_types: ["Authority"],

// src/schema/frame.ts

The Frame entity, its versioned snapshot, mode/flavor enums, jurisdiction value object, position table, and the LLM-settings projection types. // src/schema/session.ts

The "options box" framework per Article II § 3. SatisfactionPolicy is AND-by-default with at most one explicit AnyOf group; no nesting, no NOT (negations are explicit conditions like not_contradicted, not_distinguished, not_foreclosed). The required field is named all_of, parallel to any_of. The Condition type is a discriminated union with kind as the discriminator (parameterless conditions still carry a kind field; the parameterized premise_kind_in and burden_met add their own param fields). The 12-condition library is closed at v1; adding conditions requires a Stream B amendment per Stream B B6 governance.

export type { BurdenLevel };

export type Condition =
  // Structural
  | { kind: "premise_attached" }
  | { kind: "interpretation_selected" }
  | { kind: "all_children_resolved" }
  | { kind: "path_complete" }

  // Quality
  | { kind: "not_contradicted" }
  | { kind: "premise_kind_in"; kinds: string[] }
  | { kind: "burden_met"; level: BurdenLevel }

  // Authority (legal-only; offered as opt-in for academic flavor)
  | { kind: "authority_required" }
  | { kind: "authority_binding" }
  | { kind: "not_distinguished" }

  // Procedural
  | { kind: "standard_of_review_applied" }

  // Universal
  | { kind: "not_foreclosed" };

export interface SatisfactionPolicy {
  all_of: Condition[];                         // required; AND-combined; may be empty
  any_of?: Condition[];                        // optional; max one AnyOf group; no nesting (per A3)
}

      { kind: "burden_met", level: "preponderance" },
      { kind: "not_foreclosed" },
    ],
  },
  Interpretation: {
    all_of: [{ kind: "interpretation_selected" }, { kind: "not_foreclosed" }],
  },
  Term: {
    all_of: [{ kind: "interpretation_selected" }, { kind: "not_foreclosed" }],
  },
  SubQuestion: {
    all_of: [{ kind: "all_children_resolved" }, { kind: "not_foreclosed" }],
  },
  RootQuestion: {
    all_of: [{ kind: "all_children_resolved" }, { kind: "not_foreclosed" }],
  },
  Conclusion: {
    all_of: [{ kind: "path_complete" }, { kind: "not_foreclosed" }],
  },
  LogicalGate: {
    // gate_type semantics drive the rest; the frame default carries only not_foreclosed
    all_of: [{ kind: "not_foreclosed" }],
  },

// Override resolution: per A3, the per-instance options_box (if present on Checkpoint,
// RootQuestion, SubQuestion, or Interpretation) replaces the frame-default policy entirely.
// Merging is intentionally NOT supported. This function is the single point of resolution.
export function resolveEffectivePolicy(
  node_type: keyof typeof DEFAULT_SATISFACTION_POLICIES,
  per_instance: SatisfactionPolicy | undefined,
  frame_default: SatisfactionPolicy | undefined,
): SatisfactionPolicy {
  if (per_instance) return per_instance;
  if (frame_default) return frame_default;
  return DEFAULT_SATISFACTION_POLICIES[node_type];

The full validation-rule registry. Every rule is a ValidationRule instance carrying its stable id (e.g., V-FR-1), its severity (error blocks Frame → Argument transition; warning advises only), its description, and an evaluate function returning ValidationResult[]. The aggregator runValidation() walks the registry in id order (lexicographic on the rule id, which is deterministic given the fixed naming convention) and concatenates results. Stream H wires this into the C5 frame-completeness check; Stream D D3 consults it for mode-transition gating.

export interface ValidationResult {
  rule_id: string;
  severity: Severity;
  node_id?: NodeRef;
  edge_id?: EdgeRef;
  message: string;
}

export interface ValidationRule {
  id: string;                                  // e.g., "V-FR-1"
  severity: Severity;
  description: string;
  evaluate(frame: FrameVersion, session?: ArgumentSession): ValidationResult[];
}

// Aggregator. Walks rules in lexicographic id order for determinism.
export function runValidation(
  frame: FrameVersion,
  session?: ArgumentSession,
  rules: ValidationRule[] = VALIDATION_RULES,
): ValidationResult[] {
  const sorted = [...rules].sort((a, b) => a.id.localeCompare(b.id));
  const out: ValidationResult[] = [];
  for (const r of sorted) out.push(...r.evaluate(frame, session));
  return out;
}

// src/schema/migrations.ts

The schema-version migration mechanism. v1 ships at CURRENT_SCHEMA_VERSION = 1 with an empty migrations table — there is nothing to migrate from at v1. The mechanism is in place so future schema bumps walk a path of pure (unknown) => unknown functions chained from the document's schema_version to CURRENT_SCHEMA_VERSION. Each migration is required to be referentially transparent (no clock reads, no environment branches) so that exported documents migrate identically across machines and runs.

export type Migration = (input: unknown) => unknown;

export interface MigrationRegistry {
  readonly current_schema_version: number;
  // Map from source version → migration that produces source+1.
  // Walking the registry from `from` to `current_schema_version` chains the migrations.
  migrations: { [from: number]: Migration };
}

export const CURRENT_SCHEMA_VERSION = 1;

export const MIGRATION_REGISTRY: MigrationRegistry = {
  current_schema_version: CURRENT_SCHEMA_VERSION,
  migrations: {
    // empty at v1; future entries: 1: migrate_v1_to_v2, 2: migrate_v2_to_v3, ...
  },
};

export class UnknownSchemaVersionError extends Error {
  constructor(public readonly seen: number) {
    super(`Unknown schema_version ${seen}; no migration path to ${CURRENT_SCHEMA_VERSION}`);
    this.name = "UnknownSchemaVersionError";
  }
}

/**
 * Migrate an envelope of unknown shape carrying schema_version up to CURRENT_SCHEMA_VERSION.
 * Returns the migrated envelope. Throws UnknownSchemaVersionError when no path exists.
 *
 * Pure: no clock reads, no global state. The registry is the only source of behavior.
 */
export function migrate(
  envelope: { schema_version: number } & Record<string, unknown>,
  registry: MigrationRegistry = MIGRATION_REGISTRY,
): unknown {
  if (envelope.schema_version === registry.current_schema_version) return envelope;
  if (envelope.schema_version > registry.current_schema_version) {
    throw new UnknownSchemaVersionError(envelope.schema_version);
  }
  let current: unknown = envelope;
  let v = envelope.schema_version;
  while (v < registry.current_schema_version) {
    const step = registry.migrations[v];
    if (!step) throw new UnknownSchemaVersionError(envelope.schema_version);
    current = step(current);
    v += 1;
  }
  // Stamp the new schema_version onto the result.
  if (typeof current === "object" && current !== null) {
    (current as Record<string, unknown>).schema_version = registry.current_schema_version;
  }
  return current;
}

Mode-aware schema behavior summary

This section restates the mode/flavor surface visibility from current_state.html "Mode-specific surface visibility" specifically as it affects the schema layer. It is a reading aid for the coding session implementing schema; it does not introduce new commitments.

Legal mode

The following fields are active at the schema layer in legal mode (carry runtime meaning) and either ignored or absent in general mode: Frame.jurisdiction_default, SubQuestion.is_jurisdictional, RootQuestion.standard_of_review, SubQuestion.standard_of_review, Interpretation.jurisdiction_override, Checkpoint.requires_authority, Checkpoint.burden_level, Authority.{court, year, holding_summary, is_holding, is_binding, jurisdiction, binding_in}, Conclusion.remedy, the { kind: "legal" } branch of ConclusionDirection. EdgeTypes BINDING_IN (as the embedded relation) and DISTINGUISHED_BY are active. Legal-only Conditions in the satisfaction-policy library: authority_required, authority_binding, not_distinguished, standard_of_review_applied. Validation rules V-FR-7, V-FR-11, V-ARG-2 are active. Premise kind vocabulary: PREMISE_KIND_VOCABULARIES.legal.

General mode

All legal-only fields are present on the schema (the schema is mode-agnostic at the type level — fields don't disappear from Node shapes when the frame switches mode) but are runtime-inactive: validators ignore them, the runtime does not consult them, and the editor hides them. Frame.flavor is required (V-NODE-2 enforces). Frame.positions is required when the frame has Conclusions (V-FR-10). Conclusion.direction uses the { kind: "general", position_id } branch.

Personal flavor

Authority is hidden by default in the editor (per-frame opt-in to enable); when enabled, the UI label is "Source." The schema name remains Authority. Legal Authority fields stay hidden (and absent at write-time). Premise kind vocabulary: PREMISE_KIND_VOCABULARIES.general_personal. CITES is visible only when Authority is enabled. DISTINGUISHED_BY is hidden. The authority_required condition is not offered in the options-box editor.

Academic flavor

Authority is visible by default with the academic field set (author, venue, position); label "Authority." Premise kind vocabulary: PREMISE_KIND_VOCABULARIES.general_academic. CITES and DISTINGUISHED_BY both visible. The authority_required condition is offered as opt-in. Burden-level UI relabels for idiom (no schema change; see Stream D D5).

Schema-layer guarantee

The schema is a superset of every mode's active fields. Switching a frame's mode never breaks a schema-valid document: legal-only fields persist as inactive metadata, and reactivate on switching back. The validators surface mode mismatches as warnings at the schema layer (e.g., a legal-only field set on a general-mode frame is a soft warning, not a hard error) so frames don't break when their mode is changed. Hard mode-transition gating lives at the Stream D D3 layer (attemptTransition in @/modes) and operates on the Conclusion.direction mismatch specifically.

import { describe, it, expect } from "vitest";
import {
  isCheckpoint, isLogicalGate, isAndGate, isIfThenGate, isUnlessGate,
  isBooleanEvaluable,
  type Node, type LogicalGate, type Edge, type Condition,
} from "@/schema";

describe("schema/type-narrowing", () => {
  it("Node type guards narrow to the per-type interface", () => {
    function example(n: Node) {
      if (isCheckpoint(n)) {
        // n is Checkpoint here; the answer_type field is accessible.
        const at: "boolean" | "multiple_choice" | "graded" = n.answer_type;
        return at;
      }
      if (isLogicalGate(n)) {
        // n is LogicalGate here. Sub-narrow on gate_type:
        if (isAndGate(n))    { const _x: import("@/schema").NodeRef[] = n.inputs; return _x; }
        if (isIfThenGate(n)) { const _a: import("@/schema").NodeRef = n.antecedent; return _a; }
        if (isUnlessGate(n)) { const _m: import("@/schema").NodeRef = n.main; return _m; }
      }
      return null;
    }
    expect(typeof example).toBe("function");
  });

  it("LogicalGate sub-discriminator exhausts via switch on gate_type", () => {
    function exhaust(g: LogicalGate): string {
      switch (g.gate_type) {
        case "AND":     return `and-${g.inputs.length}`;
        case "OR":      return `or-${g.inputs.length}`;
        case "NOT":     return `not-${g.input}`;
        case "IF_THEN": return `if-${g.antecedent}-${g.consequent}`;
        case "UNLESS":  return `unless-${g.main}-${g.exception}`;
        // No `default` — exhaustiveness is assured by the union.
      }
    }
    expect(typeof exhaust).toBe("function");
  });

  it("Edge discriminator exhausts via switch on type", () => {
    function exhaust(e: Edge): string {

        case "DECOMPOSES_INTO":
        case "TURNS_ON":

Architectural commitments (Article II)

  • § 1 Frame/Argument Split.
