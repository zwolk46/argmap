Per-module spec for src/persistence/  ·  Stream I session I.4  ·  May 10, 2026

This document is the implementation source of truth for src/persistence/. It commits per-file TypeScript signatures, the IndexedDB store layout (declared via Dexie), the Repository interface as the only surface application code sees, the four single-transaction composite operations, the AutosaveController with 5s/30s debounce and milestone forcing, the pure diff functions for frame and session versions, the precomputed search index, the cross-tab CrossTabBus, the schema-migration plumbing, and Vitest acceptance tests for the module. The cross-module surface (what other modules import from @/persistence) is pinned in stream_i_contracts_v1.html Contract 3; this spec adds the per-file organization, internal types, atomicity contracts, and behavioral contracts the coding session implements against.

The persistence layer sits outside the runtime determinism boundary (Article II § 2 binds the runtime, not the persistence layer). src/persistence/ may read the clock for created_at / updated_at timestamps, may call crypto.randomUUID() for new entity ids, and may use Dexie's async APIs. What it must not do is leak any of those non-determinism sources into the runtime: the timestamps and ids the persistence layer writes onto FrameVersion and ArgumentSessionVersion records are inputs to compute only as fixed snapshot fields, never as live reads inside the pipeline (Stream H "Determinism implementation contract"; runtime spec self-review § 2). The persistence layer also imports only @/schema from the application codebase — not @/runtime, not @/state, not @/modes, not @/ui, not @/llm-hooks, not @/layout — keeping the dependency graph one-directional (Contract section above the contracts table).

Cross-document resolution notes (for self-review)

While compiling this spec from stream_h_persistence_v1.html against stream_i_contracts_v1.html Contract 3 and stream_i_schema_spec_v1.html, four precision questions arose where the contracts doc paraphrased or under-specified the canonical sources. In each case this spec adopts the constitutionally clean resolution; the contracts doc is the doc that needs the additive correction, not the schema, the runtime, or this persistence spec. The four items are itemized in this session's self-review (Article VIII § 4) and are logged as a single in-session-escalated flag for a future I.1 review session to bump the contracts doc — same family as F-003 and the I.3 pending flag held in flag_pending_from_i3.html. The four resolutions:

  1. Import boundary tightened: persistence imports @/schema only. Contract 3's preamble line ("persistence imports schema and runtime — the latter only to call compute on session restore") is not implemented by any persistence method in Stream H. The restore operation in Stream H H3 creates a new version from an ancestor inside a single Dexie transaction; it does not call compute. Recomputation happens at the state-layer subscription site after the restore returns. Tightening the boundary reinforces Article II § 2 (the runtime is pure; preventing persistence from importing it removes a back-door where future code could accidentally chain a clock-reading wrapper through it) and matches every actually-existing call site in the design.
  2. AutosaveController takes payload-passing signatures, not (kind, id). Contract 3's schedule(kind, id) is underspecified: the controller cannot reach into @/state for the latest entity without crossing the dependency boundary, and cannot read it from the Repository because the Repository returns last-saved state rather than the in-memory working state. The constitutionally clean resolution is for callers to hand the controller the latest PendingFrameSave / PendingSessionSave payload directly; multiple rapid calls for the same entity overwrite earlier pending payloads. The controller becomes a dumb storage-plus-timer abstraction; the state layer remains the source of truth for what to save. This refinement does not change the contract's outward purpose (debounced batch writes); it makes the signature actually implementable within the dependency graph the contracts doc itself committed.
  3. StructuralDiff is split into StructuralDiff (frame) and SessionStructuralDiff (session), and edited entries carry fields_changed: string[]. Contract 3's single StructuralDiff shape names nodes and edges collections, which fit frame versions but have no natural fill for session versions (whose collections are premises, argument_edges, checkpoint_responses, session_authorities, interpretation_selections). Two specialized types map cleanly to Stream H's diff algorithm and Stream E E5's Compare view. The fields_changed extension is additive: it lets the UI display field-level edits without re-running per-field comparison itself. The layout_only: boolean and a new layout_changed_count: number are retained as top-level signals matching Stream H's layout-bucketing UX refinement.
  4. A test-time dependency on fake-indexeddb is added. Vitest runs in Node; Node has no native IndexedDB. The persistence module's unit-test surface (round-trip, atomicity, autosave, diff, search-index, broadcast) is meaningful only against an actual IndexedDB implementation. The standard solution is fake-indexeddb as a devDependency; Vite's tree-shaking excludes it from the production bundle, and the buildkit index's "no additional dependency" rule is meant to apply to runtime/production dependencies (the rule's rationale, anchored in Article II § 2, is about runtime imports). The alternative (Playwright-only persistence tests against a real browser IndexedDB) loses unit-test feedback and slows the development loop without commensurate benefit. Logged for visibility, not because the constitutional analysis is uncertain.

export type Unsubscribe = () => void;

export interface AutosaveController {
  /**
   * Enqueue a frame save. Overwrites any prior pending payload for the same
   * Frame id (the controller flushes only the latest). Honors the 5s idle /
   * 30s max debounce.
   */
  scheduleFrameSave(payload: PendingFrameSave): void;
  scheduleSessionSave(payload: PendingSessionSave): void;
  /**
   * Enqueue an app-state save. Debounced separately at 1s; never produces
   * a SaveEvent with version_id since app-state is not versioned.
   */
  scheduleAppStateSave(state: AppState): void;

  /**
   * Force-flush pending saves immediately. Returns when all pending writes
   * have either succeeded or emitted save_failed. Used before navigation,
   * exports, milestone operations, and in beforeunload.
   */
  flushAll(): Promise<void>;
  flushFrame(frame_id: FrameId): Promise<void>;
  flushSession(session_id: SessionId): Promise<void>;
  flushAppState(): Promise<void>;

  /**
   * Force-save as milestone, bypassing the debounce. The caller is responsible
   * for setting payload.new_version.is_milestone = true.
   */
  saveFrameMilestone(payload: PendingFrameSave): Promise<void>;
  saveSessionMilestone(payload: PendingSessionSave): Promise<void>;

  on(event: AutosaveEvent, listener: (info: SaveEvent) => void): Unsubscribe;

  /** Stop all timers and detach listeners. */
  dispose(): void;
}

The Dexie database class. Stores match Stream H H1's IndexedDB store layout 1:1. Two versioning systems coexist: Dexie's database version (controls store layout — adding stores, adding indexes, changing keypaths) and the application schema_version (controls record-content shape, dispatched through @/schema's migrate()). v1 ships both at version 1 with an empty migration path. /**
 * Apply the application-level schema migration in-place across every store.
 * Called by IndexedDbRepository.openOrUpgrade() when the app starts and the
 * stored AppState.last_known_schema_version is less than CURRENT_SCHEMA_VERSION.
 *
 * Runs inside a single Dexie 'rw' transaction across all stores so partial
 * failure leaves the database untouched.
 */
export async function applySchemaMigrationSweep(
  db: ArgmapDb,
  from_version: number,
): Promise<void> {
  if (from_version === CURRENT_SCHEMA_VERSION) return;
  await db.transaction(
    "rw",
    [db.frames, db.frame_versions, db.argument_sessions, db.argument_session_versions, db.app_state],
    async () => {
      // For each store, read every record, wrap in a synthetic envelope with the
      // stored schema_version, call schemaMigrate to bring it to CURRENT_SCHEMA_VERSION,
      // and write back. v1 has empty migrations so this loop is a no-op; the structure
      // is in place for future bumps.
      const stores: Table<unknown, unknown>[] = [
        db.frames, db.frame_versions, db.argument_sessions, db.argument_session_versions,
      ];
      for (const store of stores) {
        const records = await store.toArray();
        for (const record of records) {
          const envelope = { schema_version: from_version, payload: record };
          const migrated = schemaMigrate(envelope as { schema_version: number } & Record<string, unknown>);
          // The synthetic envelope round-trips by convention: payload is the record.
          const next = (migrated as { payload?: unknown }).payload ?? migrated;
          // Replace in place; primary key is stable.
          await store.put(next as never);
        }
      }
      const app_state = (await db.app_state.get("current")) ?? null;
      if (app_state) {
        app_state.last_known_schema_version = CURRENT_SCHEMA_VERSION;
        await db.app_state.put(app_state);
      }
    },
  );
}

Two clarifications on the Dexie schema. First, the app_state singleton is keyed by the literal string "current"; the table type intersects AppState with { key: "current" } so Dexie's primary-key lookup compiles. Second, the search_index table is keyed by frame_id (one entry per Frame, not per token), matching Stream H's "precomputed token index per Frame, rebuilt on every Frame save" strategy. The per-token reverse index lives inside the entry as the token_fields map, used by searchFrames to populate FrameSearchHit.hit_field.

The concrete Repository implementation. Owns an ArgmapDb and routes every method through Dexie's transaction API. Every method that touches more than one record uses an explicit db.transaction(...) block so atomicity is enforced rather than hoped for. The constructor accepts optional now() and generateId() injections to support deterministic testing.

export class IndexedDbRepository implements Repository {
  private readonly db: ArgmapDb;
  private readonly now: () => string;
  private readonly generateId: () => string;
  private readonly search_cache: SearchIndex;

  constructor(opts: IndexedDbRepositoryOptions = {}) {
    this.db = new ArgmapDb(opts.db_name);
    this.now = opts.now ?? (() => new Date().toISOString());
    this.generateId = opts.generateId ?? (() => crypto.randomUUID());
    this.search_cache = new SearchIndex();
  }

  /**
   * Must be awaited before any other method runs. Opens the database, ensures the
   * AppState singleton exists, and runs a schema-migration sweep if needed.
   */
  async openOrUpgrade(): Promise<void> {
    try {
      await this.db.open();
    } catch (e) {
      throw new RepositoryError("openOrUpgrade", (e as Error).message);
    }
    let app_state = await this.db.app_state.get("current");
    if (!app_state) {
      app_state = {
        key: "current",
        recents: [],
        pinned: [],
        default_output_view: "path_overlay",
        side_panel_collapsed: {},
        coachmark_dismissals: {},
        last_known_schema_version: CURRENT_SCHEMA_VERSION,
      } satisfies AppState & { key: "current" };
      await this.db.app_state.put(app_state);
    } else if (app_state.last_known_schema_version < CURRENT_SCHEMA_VERSION) {
      await applySchemaMigrationSweep(this.db, app_state.last_known_schema_version);
    }
    // Warm the search cache from disk on open. Cheap at v1's scale.
    await this.search_cache.loadFromDb(this.db);
  }

  // ----- Frames -----

  async listFrames(): Promise<FrameSummary[]> {
    const all = await this.db.frames.toArray();
    return all.map(f => this.toFrameSummary(f));
  }

  async loadFrame(frame_id: FrameId): Promise<Frame> {
    const f = await this.db.frames.get(frame_id);
    if (!f) throw new RepositoryError("loadFrame", `Frame not found: ${frame_id}`);
    return f;
  }

  async saveFrame(frame: Frame): Promise<void> {
    return this.runTx(
      "saveFrame",
      "rw",
      [this.db.frames, this.db.frame_versions, this.db.search_index],
      async () => {
        await this.db.frames.put(frame);
        // The search index for this frame is regenerated when a FrameVersion saves
        // (since the indexable text lives on the version's nodes); saving the
        // Frame entity alone may have changed only Frame.title/description/tags,
        // which are also indexed — so rebuild from the current_version on disk.
        const current = await this.db.frame_versions.get(frame.current_version_id);
        if (current) {
          const entry = buildSearchIndexEntry(frame, current);
          await this.db.search_index.put(entry);
          this.search_cache.upsert(entry);
        }
      },
    );
  }

  async deleteFrame(frame_id: FrameId): Promise<void> {
    return this.runTx(
      "deleteFrame",
      "rw",
      [this.db.frames, this.db.frame_versions, this.db.argument_sessions,
       this.db.argument_session_versions, this.db.search_index],
      async () => {
        const versions = await this.db.frame_versions.where("frame_id").equals(frame_id).primaryKeys();
        const sessions = await this.db.argument_sessions.where("frame_id").equals(frame_id).primaryKeys();
        for (const sid of sessions as SessionId[]) {
          const svs = await this.db.argument_session_versions.where("session_id").equals(sid).primaryKeys();
          await this.db.argument_session_versions.bulkDelete(svs as SessionVersionId[]);
        }
        await this.db.argument_sessions.bulkDelete(sessions as SessionId[]);
        await this.db.frame_versions.bulkDelete(versions as FrameVersionId[]);
        await this.db.frames.delete(frame_id);
        await this.db.search_index.delete(frame_id);
        this.search_cache.remove(frame_id);
      },
    );
  }

  // ----- FrameVersions -----

  async listFrameVersions(frame_id: FrameId): Promise<FrameVersion[]> {
    return this.db.frame_versions
      .where("[frame_id+version_number]")
      .between([frame_id, 0], [frame_id, Infinity])
      .toArray();
  }

  async listFrameVersionSummaries(frame_id: FrameId): Promise<FrameVersionSummary[]> {
    const versions = await this.listFrameVersions(frame_id);
    return versions.map(v => ({
      id: v.id,
      frame_id: v.frame_id,
      version_number: v.version_number,
      parent_version_id: v.parent_version_id,
      created_at: v.created_at,
      is_milestone: v.is_milestone,
      change_summary: v.change_summary,

  }

  async loadFrameVersion(version_id: FrameVersionId): Promise<FrameVersion> {
    const v = await this.db.frame_versions.get(version_id);
    if (!v) throw new RepositoryError("loadFrameVersion", `FrameVersion not found: ${version_id}`);
    return v;
  }

  async saveFrameVersion(version: FrameVersion): Promise<void> {
    return this.runTx(
      "saveFrameVersion",
      "rw",
      [this.db.frames, this.db.frame_versions, this.db.search_index],

        const frame = await this.db.frames.get(version.frame_id);
        if (!frame) {
          throw new RepositoryError("saveFrameVersion",
            `Parent Frame missing: ${version.frame_id}`);
        }
        await this.db.frame_versions.put(version);
        frame.current_version_id = version.id;
        frame.updated_at = this.now();
        await this.db.frames.put(frame);
        const entry = buildSearchIndexEntry(frame, version);
        await this.db.search_index.put(entry);
        this.search_cache.upsert(entry);
      },
    );
  }

  // ----- Argument sessions -----

  async listSessionsForFrame(frame_id: FrameId): Promise<ArgumentSessionSummary[]> {
    const sessions = await this.db.argument_sessions.where("frame_id").equals(frame_id).toArray();
    const frame = await this.db.frames.get(frame_id);

  async deleteSession(session_id: SessionId): Promise<void> {

      app_version: CURRENT_APP_VERSION,

  async importFrame(envelope: FrameExport): Promise<Frame> {
    const migrated = schemaMigrate(
      envelope as unknown as { schema_version: number } & Record<string, unknown>,
    ) as FrameExport;
    const validation = runValidation(migrated.current_version);
    const errors = validation.filter(r => r.severity === "error");
    if (errors.length > 0) {
      throw new RepositoryError("importFrame",
        `Validation failed: ${errors.map(e => e.rule_id).join(", ")}`);
    }
    return this.runTx(
      "importFrame",
      "rw",
      [this.db.frames, this.db.frame_versions, this.db.search_index],
      async () => {
        await this.db.frame_versions.put(migrated.current_version);
        if (migrated.history) {
          await this.db.frame_versions.bulkPut(migrated.history);
        }
        await this.db.frames.put(migrated.frame);
        const entry = buildSearchIndexEntry(migrated.frame, migrated.current_version);
        await this.db.search_index.put(entry);
        this.search_cache.upsert(entry);
        return migrated.frame;
      },
    );
  }

  async importSession(envelope: ArgumentSessionExport): Promise<ArgumentSession> {
    const migrated = schemaMigrate(
      envelope as unknown as { schema_version: number } & Record<string, unknown>,
    ) as ArgumentSessionExport;
    return this.runTx(
      "importSession",
      "rw",
      [this.db.argument_sessions, this.db.argument_session_versions,
       this.db.frames, this.db.frame_versions, this.db.search_index],
      async () => {
        if (migrated.embedded_frame_export) {
          await this.importFrameInTx(migrated.embedded_frame_export);
        }
        await this.db.argument_session_versions.put(migrated.current_version);
        if (migrated.history) {
          await this.db.argument_session_versions.bulkPut(migrated.history);
        }
        await this.db.argument_sessions.put(migrated.session);
        return migrated.session;
      },
    );
  }

  private async importFrameInTx(envelope: FrameExport): Promise<void> { /* see implementation note below */ }

  private toFrameSummary(f: Frame): FrameSummary {
    return {
      id: f.id, title: f.title, mode: f.mode, flavor: f.flavor,
      tags: f.tags, pinned: f.pinned,
      updated_at: f.updated_at, last_opened_at: f.last_opened_at,
      current_version_id: f.current_version_id,
    };
  }

  private toSessionSummary(s: ArgumentSession, parent_frame: Frame | undefined): ArgumentSessionSummary {
    const drift_warning =
      parent_frame && parent_frame.current_version_id !== s.frame_version_id
        ? `Frame has moved forward since this session was active (frame v${parent_frame.current_version_id.slice(0, 8)} vs session v${s.frame_version_id.slice(0, 8)}).`
        : undefined;
    return {
      id: s.id, frame_id: s.frame_id, title: s.title ?? "Untitled session",
      updated_at: s.updated_at, current_version_id: s.current_version_id,
      ...(drift_warning ? { frame_version_drift_warning: drift_warning } : {}),
    };
  }

  /**
   * Wrap a Dexie transaction with consistent error mapping. Catches
   * DOMException 'QuotaExceededError' and rethrows as the typed error;
   * wraps other failures in RepositoryError.
   */
  private async runTx<T>(
    op: string,
    mode: "r" | "rw",
    tables: Table<unknown, unknown>[],
    body: () => Promise<T>,
  ): Promise<T> {
    try {
      return await this.db.transaction(mode, tables, body);
    } catch (e) {
      const err = e as Error & { name?: string };
      if (err.name === "QuotaExceededError"
        || err.name === "AbortError"
        && /quota/i.test(err.message ?? "")) {
        throw new QuotaExceededError(err.message);
      }
      if (err instanceof QuotaExceededError || err instanceof RepositoryError) {
        throw err;
      }
      throw new RepositoryError(op, err.message);
    }
  }

  /** Close the underlying database. Useful for test teardown. */
  close(): void { this.db.close(); }
}

// src/persistence/autosave.ts

The AutosaveController implementation. Stores pending payloads in two maps (frames and sessions, keyed by entity id) so multiple rapid edits to the same entity collapse to the latest payload. Two timers per entity: an idle timer (5s) reset on every schedule call, and a max-elapsed timer (30s) started on the first schedule call after a flush. Whichever fires first calls flush(entity_id). App-state saves run on a separate, simpler debouncer (1s, single shared timer because AppState is a singleton).

import {
  type AutosaveController,
  type AutosaveEvent,
  type PendingFrameSave,
  type PendingSessionSave,
  type SaveEvent,
  type Unsubscribe,
  type Repository,
  type AppState,
  QuotaExceededError,
  RepositoryError,
} from "./repository";
import type { FrameId, SessionId } from "@/schema";

export const AUTOSAVE_IDLE_MS = 5_000;
export const AUTOSAVE_MAX_MS = 30_000;
export const APP_STATE_DEBOUNCE_MS = 1_000;

interface FrameSlot {
  payload: PendingFrameSave;
  idle_timer: ReturnType<typeof setTimeout> | null;
  max_timer: ReturnType<typeof setTimeout> | null;
  /** Set while a save is in flight; further schedules queue and run on completion. */
  in_flight: boolean;
}
interface SessionSlot {
  payload: PendingSessionSave;
  idle_timer: ReturnType<typeof setTimeout> | null;
  max_timer: ReturnType<typeof setTimeout> | null;
  in_flight: boolean;
}

export interface AutosaveControllerOptions {
  /** The Repository to write through. */
  repo: Repository;
  /**
   * Optional clock injector for tests. The controller itself does not read the clock;
   * it only uses setTimeout for debouncing. Provided here for symmetry with
   * IndexedDbRepository's injection surface.
   */
  setTimeoutFn?: typeof setTimeout;
  clearTimeoutFn?: typeof clearTimeout;
}

export function createAutosaveController(opts: AutosaveControllerOptions): AutosaveController {
  return new AutosaveControllerImpl(opts);
}

class AutosaveControllerImpl implements AutosaveController {
  private readonly repo: Repository;
  private readonly setTimeoutFn: typeof setTimeout;
  private readonly clearTimeoutFn: typeof clearTimeout;
  private readonly frame_slots = new Map<FrameId, FrameSlot>();
  private readonly session_slots = new Map<SessionId, SessionSlot>();
  private app_state_payload: AppState | null = null;
  private app_state_timer: ReturnType<typeof setTimeout> | null = null;
  private app_state_in_flight = false;
  private readonly listeners: { [E in AutosaveEvent]: Set<(e: SaveEvent) => void> } = {
    save_succeeded: new Set(),
    save_failed: new Set(),
  };
  private disposed = false;

  constructor(opts: AutosaveControllerOptions) {
    this.repo = opts.repo;
    this.setTimeoutFn = opts.setTimeoutFn ?? setTimeout;
    this.clearTimeoutFn = opts.clearTimeoutFn ?? clearTimeout;
  }

  scheduleFrameSave(payload: PendingFrameSave): void {
    if (this.disposed) return;
    const id = payload.frame.id;
    const slot = this.frame_slots.get(id) ?? {
      payload, idle_timer: null, max_timer: null, in_flight: false,
    };
    slot.payload = payload;             // overwrite-with-latest
    if (slot.idle_timer) this.clearTimeoutFn(slot.idle_timer);
    slot.idle_timer = this.setTimeoutFn(() => { void this.flushFrame(id); }, AUTOSAVE_IDLE_MS);
    if (!slot.max_timer) {
      slot.max_timer = this.setTimeoutFn(() => { void this.flushFrame(id); }, AUTOSAVE_MAX_MS);
    }
    this.frame_slots.set(id, slot);
  }

  scheduleSessionSave(payload: PendingSessionSave): void {
    if (this.disposed) return;
    const id = payload.session.id;
    const slot = this.session_slots.get(id) ?? {
      payload, idle_timer: null, max_timer: null, in_flight: false,
    };
    slot.payload = payload;
    if (slot.idle_timer) this.clearTimeoutFn(slot.idle_timer);
    slot.idle_timer = this.setTimeoutFn(() => { void this.flushSession(id); }, AUTOSAVE_IDLE_MS);
    if (!slot.max_timer) {
      slot.max_timer = this.setTimeoutFn(() => { void this.flushSession(id); }, AUTOSAVE_MAX_MS);
    }
    this.session_slots.set(id, slot);
  }

  scheduleAppStateSave(state: AppState): void {
    if (this.disposed) return;
    this.app_state_payload = state;
    if (this.app_state_timer) this.clearTimeoutFn(this.app_state_timer);
    this.app_state_timer = this.setTimeoutFn(() => { void this.flushAppState(); }, APP_STATE_DEBOUNCE_MS);
  }

  async flushAll(): Promise<void> {
    const frame_ids = Array.from(this.frame_slots.keys()).sort();
    const session_ids = Array.from(this.session_slots.keys()).sort();
    const tasks: Promise<void>[] = [];
    for (const id of frame_ids) tasks.push(this.flushFrame(id));
    for (const id of session_ids) tasks.push(this.flushSession(id));
    if (this.app_state_payload) tasks.push(this.flushAppState());
    await Promise.allSettled(tasks);
  }

  async flushFrame(frame_id: FrameId): Promise<void> {
    const slot = this.frame_slots.get(frame_id);
    if (!slot) return;
    if (slot.in_flight) {
      // Queue: leave the latest payload in slot; the running save will re-fire
      // on completion if a newer payload landed.
      return;
    }
    if (slot.idle_timer) { this.clearTimeoutFn(slot.idle_timer); slot.idle_timer = null; }
    if (slot.max_timer)  { this.clearTimeoutFn(slot.max_timer);  slot.max_timer = null;  }
    const payload = slot.payload;
    slot.in_flight = true;
    try {
      const versioned = this.applyChangeSummary(payload);
      await this.repo.saveFrameVersion(versioned.new_version);
      // saveFrameVersion updates Frame.current_version_id + updated_at atomically;
      // an additional saveFrame(payload.frame) is unnecessary for the version-save
      // path. The Frame's other fields (title, tags, etc.) are also versioned-with
      // -the-version per Stream H decision #6, so no separate Frame.put is needed.
      this.emit("save_succeeded", {
        kind: "frame", id: frame_id, version_id: versioned.new_version.id,
      });
      this.frame_slots.delete(frame_id);
    } catch (e) {
      const err = e as Error;
      this.emit("save_failed", {
        kind: "frame", id: frame_id,
        error: err instanceof QuotaExceededError || err instanceof RepositoryError
          ? err : new RepositoryError("flushFrame", err.message),
      });
      slot.in_flight = false;
      // Retry-on-next-edit per Stream H "Retry: on the next edit"; we don't
      // reschedule here, but we leave the slot in the map so the next schedule
      // call resumes from the current payload.
    }
  }

  async flushSession(session_id: SessionId): Promise<void> {
    const slot = this.session_slots.get(session_id);
    if (!slot) return;
    if (slot.in_flight) return;
    if (slot.idle_timer) { this.clearTimeoutFn(slot.idle_timer); slot.idle_timer = null; }
    if (slot.max_timer)  { this.clearTimeoutFn(slot.max_timer);  slot.max_timer = null;  }
    const payload = slot.payload;
    slot.in_flight = true;
    try {
      const versioned = this.applySessionChangeSummary(payload);
      await this.repo.saveSessionVersion(versioned.new_version);
      this.emit("save_succeeded", {
        kind: "session", id: session_id, version_id: versioned.new_version.id,
      });
      this.session_slots.delete(session_id);
    } catch (e) {
      const err = e as Error;
      this.emit("save_failed", {
        kind: "session", id: session_id,
        error: err instanceof QuotaExceededError || err instanceof RepositoryError
          ? err : new RepositoryError("flushSession", err.message),
      });
      slot.in_flight = false;
    }
  }

  async flushAppState(): Promise<void> {
    if (!this.app_state_payload || this.app_state_in_flight) return;
    if (this.app_state_timer) { this.clearTimeoutFn(this.app_state_timer); this.app_state_timer = null; }
    const state = this.app_state_payload;
    this.app_state_payload = null;
    this.app_state_in_flight = true;
    try {
      await this.repo.saveAppState(state);
      this.emit("save_succeeded", { kind: "app_state" });
    } catch (e) {
      const err = e as Error;
      this.emit("save_failed", {
        kind: "app_state",
        error: err instanceof QuotaExceededError || err instanceof RepositoryError
          ? err : new RepositoryError("flushAppState", err.message),
      });
    } finally {
      this.app_state_in_flight = false;
    }
  }

  async saveFrameMilestone(payload: PendingFrameSave): Promise<void> {
    // The caller is responsible for setting payload.new_version.is_milestone = true.
    // We replace any pending payload for this frame and flush immediately.
    this.scheduleFrameSave(payload);
    await this.flushFrame(payload.frame.id);
  }

  async saveSessionMilestone(payload: PendingSessionSave): Promise<void> {
    this.scheduleSessionSave(payload);
    await this.flushSession(payload.session.id);
  }

  on(event: AutosaveEvent, listener: (info: SaveEvent) => void): Unsubscribe {
    this.listeners[event].add(listener);
    return () => { this.listeners[event].delete(listener); };
  }

  dispose(): void {
    this.disposed = true;
    for (const [, slot] of this.frame_slots) {
      if (slot.idle_timer) this.clearTimeoutFn(slot.idle_timer);
      if (slot.max_timer) this.clearTimeoutFn(slot.max_timer);
    }
    for (const [, slot] of this.session_slots) {
      if (slot.idle_timer) this.clearTimeoutFn(slot.idle_timer);
      if (slot.max_timer) this.clearTimeoutFn(slot.max_timer);
    }
    if (this.app_state_timer) this.clearTimeoutFn(this.app_state_timer);
    this.frame_slots.clear();
    this.session_slots.clear();
    this.listeners.save_succeeded.clear();
    this.listeners.save_failed.clear();
  }

  private applyChangeSummary(p: PendingFrameSave): PendingFrameSave {
    if (!p.change_summary) return p;
    return { ...p, new_version: { ...p.new_version, change_summary: p.change_summary } };
  }

  private applySessionChangeSummary(p: PendingSessionSave): PendingSessionSave {
    if (!p.change_summary) return p;
    return { ...p, new_version: { ...p.new_version, change_summary: p.change_summary } };
  }

  private emit(event: AutosaveEvent, info: SaveEvent): void {
    for (const listener of this.listeners[event]) {
      try { listener(info); } catch { /* listener errors are isolated */ }
    }
  }
}

Two implementation notes. First, the in-flight guard returns early without rescheduling: if a save fails, the slot remains populated with the latest payload, and the next schedule* call (which happens on the next user edit per Stream H "retry: on the next edit") re-arms the timers. Second, the controller never inspects or mutates the Frame entity beyond writing it through the Repository. The state layer is responsible for producing a fully-formed PendingFrameSave (including the new FrameVersion with its own id, created_at, parent_version_id, version_number, is_milestone flag, etc.); the controller simply queues and persists. This division keeps the controller a small, testable timer-and-storage abstraction.

Pure diff functions for FrameVersions and ArgumentSessionVersions. Per Stream H H3, the diff function is referentially transparent and runs in O(n) over the entity counts. LAYOUT_ONLY_FIELDS names the fields whose changes are bucketed out of fields_changed and into the top-level layout_changed_count. The current LAYOUT_ONLY_FIELDS list is ["presentation"] — the only top-level Node field carrying purely layout state (per the schema spec's Node.presentation?.{x, y, collapsed, color_override} nesting).

import {
  type FrameVersion, type ArgumentSessionVersion,
  type NodeRef, type EdgeRef,
} from "@/schema";
import {
  type StructuralDiff, type SessionStructuralDiff,
  type NodeEditEntry, type EdgeEditEntry, type MetadataChange,
  LAYOUT_ONLY_FIELDS,
} from "./repository";

/**
 * Pure. O(|nodes_a| + |nodes_b| + |edges_a| + |edges_b|). Output arrays are
 * sorted by id for stable test snapshots. Layout-only changes are bucketed
 * into layout_changed_count per Stream H H3.
 */
export function diffFrameVersions(a: FrameVersion, b: FrameVersion): StructuralDiff {
  const nodes_a = new Map(a.nodes.map(n => [n.id, n]));
  const nodes_b = new Map(b.nodes.map(n => [n.id, n]));
  const edges_a = new Map(a.edges.map(e => [e.id, e]));
  const edges_b = new Map(b.edges.map(e => [e.id, e]));

  const node_added: NodeRef[] = [];
  const node_removed: NodeRef[] = [];
  const node_edited: NodeEditEntry[] = [];
  let layout_changed_count = 0;
  for (const [id, after] of nodes_b) {
    const before = nodes_a.get(id);
    if (!before) { node_added.push(id); continue; }
    const all_changes = diffFields(before, after);
    if (all_changes.length === 0) continue;
    const non_layout = all_changes.filter(f => !LAYOUT_ONLY_FIELDS.includes(f as never));
    if (non_layout.length === 0) {
      layout_changed_count++;
    } else {
      node_edited.push({ id, fields_changed: non_layout });
    }
  }
  for (const id of nodes_a.keys()) {
    if (!nodes_b.has(id)) node_removed.push(id);
  }

  const edge_added: EdgeRef[] = [];
  const edge_removed: EdgeRef[] = [];
  const edge_edited: EdgeEditEntry[] = [];
  for (const [id, after] of edges_b) {
    const before = edges_a.get(id);
    if (!before) { edge_added.push(id); continue; }
    const changes = diffFields(before, after);
    if (changes.length > 0) edge_edited.push({ id, fields_changed: changes });
  }
  for (const id of edges_a.keys()) {
    if (!edges_b.has(id)) edge_removed.push(id);
  }

  const metadata_changed = diffFrameVersionMetadata(a, b);

  const layout_only =
    node_added.length === 0 && node_removed.length === 0 && node_edited.length === 0
    && edge_added.length === 0 && edge_removed.length === 0 && edge_edited.length === 0
    && metadata_changed.length === 0
    && layout_changed_count > 0;

  return {
    nodes: {
      added: node_added.sort(),
      removed: node_removed.sort(),
      edited: node_edited.sort((x, y) => x.id.localeCompare(y.id)),
    },
    edges: {
      added: edge_added.sort(),
      removed: edge_removed.sort(),
      edited: edge_edited.sort((x, y) => x.id.localeCompare(y.id)),
    },
    metadata: { changed_fields: metadata_changed },
    layout_only,
    layout_changed_count,
  };
}

/**
 * Compare two entities and return the top-level field names whose values differ
 * by deep-structural equality. Uses JSON-string equality for nested values
 * (canonical-string equality at the persistence layer is fine; runtime
 * determinism is a separate concern handled in @/runtime).
 */
function diffFields<T extends object>(before: T, after: T): string[] {
  const keys = new Set<string>([...Object.keys(before), ...Object.keys(after)]);
  const out: string[] = [];
  for (const k of keys) {
    const bv = (before as Record<string, unknown>)[k];
    const av = (after as Record<string, unknown>)[k];
    if (!deepEqual(bv, av)) out.push(k);
  }
  return out.sort();
}

function deepEqual(a: unknown, b: unknown): boolean {
  // null/undefined normalization: persistence treats absent and null as equal.
  if (a === null || a === undefined) return b === null || b === undefined;
  return JSON.stringify(a) === JSON.stringify(b);
}

function diffFrameVersionMetadata(a: FrameVersion, b: FrameVersion): MetadataChange[] {
  const fields_to_check = [
    "version_number", "parent_version_id", "is_milestone",
    "change_summary", "llm_settings_snapshot",
  ] as const;
  return fields_to_check
    .filter(f => !deepEqual(a[f], b[f]))
    .map(field => ({ field }));
}

Two notes on the diff design. First, diffFields uses JSON.stringify-string equality, which is acceptable here because the persistence layer is not the determinism boundary and because the diff is a debugging/UX surface, not a runtime input. Engine-specific JSON key-ordering differences could theoretically cause a false-positive change report, but in practice modern engines order keys by insertion which is itself stable for our records. If a future test surfaces false positives, this helper switches to a canonical-key comparator. Second, diffSessionVersions's signature is pinned here but its body is the same pattern repeated across premises, argument_edges, checkpoint_responses, session_authorities, and interpretation_selections; the coding session implements it analogously to diffFrameVersions, keying each collection by its natural id (the checkpoint_id for responses; the term_id for selections; the entity id for the others).

// src/persistence/search-index.ts

Token extraction, the in-memory cache, and the searchFrames query implementation. Stream H H1's search index strategy: precomputed token list per Frame, rebuilt on every Frame save, queried via in-memory intersection. v1 tokenization is whitespace + lowercasing + punctuation stripping; no stemming, no fuzzy match. The query is AND-intersection over query tokens against each frame's token set; matches are deduplicated by frame and surfaced as FrameSearchHits with the most informative hit_field chosen by Stream H's field priority.

import { ArgmapDb, type SearchIndexEntry } from "./dexie-schema";
import { type FrameSearchHit } from "./repository";
import {
  type Frame, type FrameVersion, type FrameId,
} from "@/schema";

/**
 * Tokenize a single input string. v1: lowercased, punctuation-stripped,
 * whitespace-split. Pure.
 */
export function tokenize(input: string): string[] {
  return input
    .toLowerCase()
    .replace(/[^\p{L}\p{N}\s]+/gu, " ")     // strip punctuation, keep letters/digits
    .split(/\s+/)
    .filter(t => t.length > 0);
}

/**
 * Build a SearchIndexEntry for one Frame + its current FrameVersion. The entry
 * is keyed by frame_id and stores both the unioned token list (for fast intersection)
 * and a per-token reverse index back to the source field (for FrameSearchHit.hit_field).
 * Pure; tests assert determinism by snapshot.
 */
export function buildSearchIndexEntry(
  frame: Frame,
  current: FrameVersion,
): SearchIndexEntry {
  const tokens = new Set<string>();
  const token_fields: { [token: string]: Array<FrameSearchHit["hit_field"]> } = {};

  const add = (text: string | undefined, field: FrameSearchHit["hit_field"]): void => {
    if (!text) return;
    for (const t of tokenize(text)) {
      tokens.add(t);
      const list = token_fields[t] ?? [];
      if (!list.includes(field)) list.push(field);
      token_fields[t] = list;
    }
  };

  add(frame.title, "title");
  add(frame.description, "description");
  for (const tag of frame.tags) add(tag, "tag");

  for (const node of current.nodes) {
    // The switch on node.type narrows `node` to the per-type interface in each
    // branch via the discriminated-union encoding in @/schema/nodes; no casts
    // are needed inside the cases.
    switch (node.type) {
      case "RootQuestion":
      case "SubQuestion":
      case "Interpretation":
        add(node.statement, "node_text");
        break;
      case "Term":
        add(node.name, "node_text");
        break;
      case "Checkpoint":
        add(node.question, "node_text");
        break;
      case "Conclusion":
        add(node.statement, "conclusion_statement");
        if (node.reasoning_summary) add(node.reasoning_summary, "node_text");
        break;
      case "Authority":
        add(node.citation, "node_text");
        break;
      case "Premise":
        // Premise is an argument-layer node (lives on ArgumentSession.premises,
        // not on FrameVersion.nodes) — this branch is unreachable when iterating
        // a FrameVersion's nodes. The case is kept for exhaustiveness; the
        // schema's `Node` discriminated union includes Premise so omitting it
        // would fail the type-narrowing check.
        break;
      case "LogicalGate":
        // gates carry no user text; skip
        break;
    }
    if (node.notes) add(node.notes, "node_text");
  }

  return {
    frame_id: frame.id,
    tokens: Array.from(tokens).sort(),    // sorted for deterministic snapshots
    token_fields,
    title: frame.title,
    built_at: frame.updated_at,
  };
}

/**
 * In-memory cache loaded from the search_index store on Repository open and
 * updated transactionally on every Frame/FrameVersion save. Query is AND-intersection
 * over query tokens; results sorted by an approximate match-count score.
 */
export class SearchIndex {
  private entries = new Map<FrameId, SearchIndexEntry>();
  private token_to_frames = new Map<string, Set<FrameId>>();

  async loadFromDb(db: ArgmapDb): Promise<void> {
    this.entries.clear();
    this.token_to_frames.clear();
    const all = await db.search_index.toArray();
    for (const entry of all) this.upsert(entry);
  }

  upsert(entry: SearchIndexEntry): void {
    // Remove old reverse-index entries for this frame, if any.
    const previous = this.entries.get(entry.frame_id);
    if (previous) {
      for (const t of previous.tokens) {
        this.token_to_frames.get(t)?.delete(entry.frame_id);
      }
    }
    this.entries.set(entry.frame_id, entry);
    for (const t of entry.tokens) {
      const set = this.token_to_frames.get(t) ?? new Set<FrameId>();
      set.add(entry.frame_id);
      this.token_to_frames.set(t, set);
    }
  }

  remove(frame_id: FrameId): void {
    const previous = this.entries.get(frame_id);
    if (!previous) return;
    for (const t of previous.tokens) {
      this.token_to_frames.get(t)?.delete(frame_id);
    }
    this.entries.delete(frame_id);
  }

  query(raw_query: string): FrameSearchHit[] {
    const query_tokens = tokenize(raw_query);
    if (query_tokens.length === 0) return [];
    // AND-intersect candidate frame ids by token.
    let candidates: Set<FrameId> | null = null;
    for (const qt of query_tokens) {
      const frames = this.token_to_frames.get(qt) ?? new Set<FrameId>();
      if (candidates === null) candidates = new Set(frames);
      else for (const id of Array.from(candidates)) {
        if (!frames.has(id)) candidates.delete(id);
      }
      if (candidates.size === 0) return [];
    }
    const hits: FrameSearchHit[] = [];
    for (const frame_id of Array.from(candidates ?? new Set<FrameId>()).sort()) {
      const entry = this.entries.get(frame_id);
      if (!entry) continue;
      // hit_field selection: priority title > conclusion_statement > tag >
      // node_text > description (most specific first, per Stream E's listing).
      const priority: Array<FrameSearchHit["hit_field"]> = [
        "title", "conclusion_statement", "tag", "node_text", "description",
      ];
      let chosen: FrameSearchHit["hit_field"] = "title";
      outer: for (const field of priority) {
        for (const qt of query_tokens) {
          if (entry.token_fields[qt]?.includes(field)) { chosen = field; break outer; }
        }
      }
      hits.push({
        frame_id,
        title: entry.title,
        hit_field: chosen,
        snippet: this.buildSnippet(entry, query_tokens),
      });
    }
    return hits;
  }

  private buildSnippet(entry: SearchIndexEntry, query_tokens: string[]): string {
    // Lightweight snippet: a few of the query tokens that matched, separated by '·'.
    // The full snippet construction with surrounding context is deferred to the UI
    // (which has both the FrameVersion in hand and can render context inline).
    const matched = query_tokens.filter(qt => entry.tokens.includes(qt));
    return matched.slice(0, 6).join(" · ");
  }
}

The snippet construction in v1 is intentionally minimal. Stream E's frame-list / search UI is expected to render proper contextual snippets by joining back to the loaded FrameVersion when displaying results; the Repository surface returns the structured hit (frame_id, title, hit_field, terse snippet) and the UI does the join. This keeps searchFrames a pure cache lookup with no heavy text manipulation, which matters when the user is typing in the search box and the cache is queried on every keystroke.

class BroadcastChannelBus implements CrossTabBus {
  private readonly channel: BroadcastChannel;
  private readonly listeners = new Map<
    keyof BroadcastEvents,
    Set<(payload: BroadcastEvents[keyof BroadcastEvents]) => void>
  >();

  constructor(channel: BroadcastChannel) {
    this.channel = channel;
    this.channel.onmessage = (e: MessageEvent) => {
      const msg = e.data as { event: keyof BroadcastEvents; payload: unknown };
      const set = this.listeners.get(msg.event);
      if (!set) return;
      for (const listener of set) {
        try { listener(msg.payload as never); } catch { /* isolate */ }
      }
    };
  }

  publish<K extends keyof BroadcastEvents>(event: K, payload: BroadcastEvents[K]): void {
    this.channel.postMessage({ event, payload });
  }

  subscribe<K extends keyof BroadcastEvents>(
    event: K,
    listener: (payload: BroadcastEvents[K]) => void,
  ): Unsubscribe {
    const set = (this.listeners.get(event) ?? new Set()) as Set<(p: BroadcastEvents[K]) => void>;
    set.add(listener);
    this.listeners.set(event, set as never);
    return () => { set.delete(listener); };
  }

  close(): void {
    this.listeners.clear();
    this.channel.close();
  }
}

  subscribe(): Unsubscribe { return () => {}; }
  close(): void { /* no-op */ }
}

Atomicity contracts

Five composite operations require single-transaction atomicity. Each is implemented inside one Dexie db.transaction(...) block (or, for saveFrameVersion, inside the per-method runTx wrapper that opens one). Either every record commits or none does. The contracts below pin the precise transaction scope, the read steps, the write steps, and the failure semantics for each. The coding session implements each per this section.

§ 1. Frame-template deep-copy (createFrameFromTemplate)

Per Stream H decision #7 and Stream D D1. Creates a new Frame from an existing one by deep-copying every node and edge with regenerated UUIDs.

  1. Read. Load the source Frame and its current_version FrameVersion.
  2. Build id translation map. Walk every Node in source.nodes; for each, allocate generateId() as the new id; record old → new. Same for every Edge.
  3. Rewrite internal references. For each node in the destination copy, rewrite every NodeRef it carries through the translation map: CheckpointOption.target_node_id, AndGate.inputs[] / OrGate.inputs[], NotGate.input, IfThenGate.antecedent / consequent, UnlessGate.main / exception, LogicalGate.output_target (any variant), Term.linked_to. For each edge in the destination copy, rewrite source and target.
  4. Construct destination entities. A new Frame with fresh id, title = new_title, all other Frame fields deep-cloned, current_version_id pointing at the new version, created_at = updated_at = now(), llm_settings initialized as { ..., invocations: [] } (the audit log starts empty in the new frame — the original frame's invocation history does not transfer because it concerns the original's specific hook commits). A new FrameVersion with fresh id, the new frame_id, version_number: 1, no parent_version_id, created_at = now(), is_milestone: true, change_summary: "Created from template: <source_title>".
  5. Write. Inside one "rw" transaction over frames, frame_versions, and search_index: put the new FrameVersion, put the new Frame, put the new SearchIndexEntry.
  6. Failure. Dexie rolls back the transaction on any error; the function rethrows the mapped error and no record persists. The source Frame is never modified.

Open design point: the per-Frame llm_settings.invocations log is reset in the template-copy (the new frame starts with no invocation history). Carrying the source's invocations would be misleading — the invocations refer to commits on nodes with old ids that no longer exist. The provenance trail for the new frame begins at its first hook invocation, if any. Stream G's hooks therefore see an empty log on the template-copied frame, which is the intended cold-start behavior.

§ 2. Session migration (migrateSession)

Per Stream H decision #7 and Stream D D2/D3. Forks a new ArgumentSessionVersion against a newer FrameVersion of the underlying frame and applies orphan resolutions.

  1. Read. Load the ArgumentSession, its current_version ArgumentSessionVersion, and the target FrameVersion.
  2. Apply orphan resolutions. For each OrphanResolution: if kind === "discard", remove every CheckpointResponse / InterpretationSelection / argument_edge that references source_node_id; if kind === "reattach", rewrite each such reference from source_node_id to target_node_id; "no_op" is a noop. The traversal is over the prior current_version's argument-layer collections, producing the new version's collections.
  3. Construct new ArgumentSessionVersion. Fresh id, session_id from the session, version_number = max(existing_versions) + 1, parent_version_id = prior_current.id, created_at = now(), is_milestone: true, change_summary: "Migrated to frame v<N>", the post-resolution argument-layer collections, and the new frame_version_id + a fresh frame_version_snapshot (the target FrameVersion's nodes/edges, deep-cloned per Article II § 1's frame/argument split).
  4. Update session. Set session.current_version_id = new_version.id, session.frame_version_id = target_frame_version_id, session.updated_at = now().
  5. Write. Inside one "rw" transaction over argument_sessions and argument_session_versions: put the new version, put the updated session.
  6. Failure. Rollback as in § 1.

§ 3. Frame version restore (restoreFrameVersion)

Per Stream H H3 restore implementation. Creates a new FrameVersion off the named ancestor; the ancestor is never modified.

  1. Read. Load the Frame and the ancestor FrameVersion. Query frame_versions for the maximum existing version_number on this frame.
  2. Construct new FrameVersion. Fresh id, the frame's frame_id, version_number = max + 1, parent_version_id = ancestor.id, created_at = now(), deep-cloned nodes and edges from the ancestor, is_milestone: true, change_summary: \`restored from version ${ancestor.version_number}\`. The llm_settings_snapshot (F-001) is also deep-cloned from the ancestor: the calibrated thresholds are preserved across the restore so that compute() produces identical output as the ancestor did.
  3. Update Frame. Set current_version_id = new_version.id, updated_at = now().
  4. Write. Inside one "rw" transaction over frames, frame_versions, and search_index: put the new version, put the updated Frame, put the rebuilt SearchIndexEntry. The search index is rebuilt because the current-version-of-truth has changed.
  5. Failure. Rollback as in § 1; the ancestor is unaffected (Article II § 4).

Version numbering is monotonic across the full frame, not per-branch. After restoring v3 of a 7-version frame, the new version is v8 with parent_version_id: v3.id. This matches Stream H H3's branch-tree rendering, which displays versions chronologically with branches drawn as indents.

§ 4. Session version restore (restoreSessionVersion)

The session-layer analog of § 3. Identical pattern over argument_sessions and argument_session_versions; deep-clones the ancestor's argument-layer collections and frame_version_snapshot; sets is_milestone: true and change_summary: \`restored from version ${ancestor.version_number}\`. Version numbering is monotonic across the full session.

§ 5. saveFrameVersion (the auto-save path)

Already in the file-level signatures above. Single transaction over frames, frame_versions, and search_index: put the new FrameVersion, update the parent Frame's current_version_id + updated_at, regenerate and put the SearchIndexEntry. The autosave path always flows through this method; the controller does not bypass it for individual record writes.

Atomicity testing

Per the Vitest acceptance test plan below, each composite operation is exercised with two cases: (a) a happy-path completion that asserts every record is present after the call, and (b) an injected mid-transaction failure (via a Dexie hook that aborts after the second write) that asserts the prior state is preserved exactly. The injected-failure tests run against fake-indexeddb's transaction model, which mirrors browser behavior.

Quota and failure handling

Per Stream H H1 "Quota and failure handling." Three failure modes, each mapped to a typed error on the Repository surface:

  • Quota exceeded on save. The Dexie transaction fails with a DOMException whose name is "QuotaExceededError". The runTx wrapper detects this and rethrows QuotaExceededError. The AutosaveController emits save_failed with the typed error; the state layer surfaces the non-blocking banner per Stream H "Failure handling" and reschedules on the next user edit.
  • Generic transaction failure. Any other transaction abort produces RepositoryError with the failed operation name. Same UI affordance.
  • Read failure on app start. A failed openOrUpgrade throws RepositoryError("openOrUpgrade", ...). The state layer catches at startup and routes to the recovery screen (per Stream H, the screen offers "Export raw data," "Open read-only," "Reset database" — those are state/UI concerns; this layer only signals).

In-memory state preservation is the caller's responsibility — the Repository never holds the only copy of pending data. The AutosaveController retains the latest PendingFrameSave / PendingSessionSave in its slot on a failed flush so the next schedule call resumes from the unsaved payload, but the Zustand stores in @/state remain the source of truth for the working copy.

Schema migration plumbing

Three places where migrations execute, all routed through @/schema's pure migrate() function (which walks the MigrationRegistry from envelope.schema_version to CURRENT_SCHEMA_VERSION with no clock reads or environment branches).

  1. On import. importFrame and importSession call schemaMigrate(envelope) before any database write. Validation runs against the migrated envelope; only validated records reach disk.
  2. On app open. openOrUpgrade reads AppState.last_known_schema_version and, if less than CURRENT_SCHEMA_VERSION, calls applySchemaMigrationSweep to walk every record in every store through schemaMigrate inside one transaction across all stores. v1 ships with CURRENT_SCHEMA_VERSION = 1 and an empty migrations table, so this is a no-op until a future bump; the structure is in place so that the future bump is mechanical.
  3. On Dexie database-version upgrade. If a future build adds a store or index, Dexie's this.version(N).stores(...).upgrade(async (tx) => { ... }) hook runs at db.open(). The upgrade callback is allowed to call schemaMigrate record-by-record for any store whose record shape also changed. The Dexie version bump and the schema_version bump are independent: a layout-only Dexie change does not require a schema_version bump, and a record-shape-only schema_version bump (no new stores or indexes) does not require a Dexie version bump.

The migration mechanism's purity is asserted at the schema layer (per the schema spec's tests/schema/round-trip.test.ts); the persistence layer adds tests that the sweep runs idempotently and that a forced bump from a hand-rolled last_known_schema_version: 0 AppState exercises the registry walk even with an empty registry.

Determinism implementation contract (persistence-side)

Stream H "Determinism implementation contract" applies; this section names the persistence-specific responsibilities and what's intentionally not the persistence layer's concern.

What the persistence layer guarantees

  • Import boundary. src/persistence/ imports only @/schema and third-party libraries (dexie; fake-indexeddb in tests only). It does not import @/runtime, @/state, @/modes, @/ui, @/llm-hooks, or @/layout. Enforced by ESLint no-restricted-imports. This is tightened from Contract 3's preamble line — see Cross-document resolution note #1.
  • Snapshot inputs to compute are deterministic. Every FrameVersion and ArgumentSessionVersion the persistence layer writes is fully self-contained (nodes/edges + the F-001 llm_settings_snapshot projection on FrameVersion + the F-002 output_overrides on ArgumentSessionVersion + the frame_version_snapshot on session). When compute reads any of these, it reads a fixed snapshot, not a live record.
  • Migrations are pure. The schema migration sweep walks records in a single Dexie transaction and applies schemaMigrate deterministically (pure function from envelope to envelope). The Dexie database version is incremented monotonically; downgrades are unsupported by design.

What the persistence layer is not responsible for

  • Output determinism. The runtime spec (I.3) carries the determinism guarantee for compute; the persistence layer only feeds it inputs. The computed_at field on ComputeInput (per the runtime spec's escalation) is supplied by the state layer when invoking compute, not by the persistence layer.
  • Clock determinism in created_at / updated_at. These fields read Date.now() at write time and are not deterministic across runs; they're informational provenance, not inputs to compute. The injected now() on IndexedDbRepositoryOptions lets tests freeze them when snapshot comparisons require it.
  • Id determinism. crypto.randomUUID() is non-deterministic by design. The injected generateId() lets tests substitute a counter-based generator when needed.

import "fake-indexeddb/auto";
import { IndexedDbRepository } from "@/persistence";

let test_db_counter = 0;

export function injectedNow(start_iso: string): () => string {
  let t = new Date(start_iso).getTime();
  return () => { const s = new Date(t).toISOString(); t += 1_000; return s; };
}

export function injectedGenerateId(): () => string {
  let i = 0;
  return () => {
    i += 1;
    return `00000000-0000-4000-8000-${i.toString(16).padStart(12, "0")}`;
  };
}

export async function freshDb(name = `test_${++test_db_counter}`): Promise<IndexedDbRepository> {
  const repo = new IndexedDbRepository({
    db_name: name,
    now: injectedNow("2026-05-10T00:00:00.000Z"),
    generateId: injectedGenerateId(),
  });
  await repo.openOrUpgrade();
  return repo;
}

Self-review (Article VIII § 4)

Architectural commitments (Article II §§ 1–6)

  • § 1 Frame/Argument Split. Honored. IndexedDB stores frames / frame_versions are entirely separate from argument_sessions / argument_session_versions; the Repository interface treats them as independent entities; the AutosaveController has separate per-id slots for each side; the diff functions are typed separately (StructuralDiff vs. SessionStructuralDiff). Each ArgumentSession carries its own frame_version_snapshot — the session's logical content is sealed against frame edits without coupling the two version streams. ✓
  • § 2 Determinism. Honored at the persistence boundary. The persistence layer reads the clock for created_at/updated_at and uses crypto.randomUUID for new ids; both are isolated behind injection points (now(), generateId()) so tests can freeze them. None of these values feed compute at runtime except as fixed snapshot fields. The import boundary (only @/schema from the application codebase) keeps the dependency graph one-directional. Schema migrations route through @/schema's pure migrate() and run inside transactions for atomicity. ✓
  • § 3 Per-Node Satisfaction Criteria. Honored. The persistence layer stores SatisfactionPolicy shapes as deeply-nested fields on Node entities; nothing here reaches into the policy structure or its evaluation. The diff function compares options_box field-by-field via the generic deepEqual; no specialized handling. ✓
  • § 4 Independent Versioning. Honored, and structurally enforced. The four version-related stores have independent primary keys, independent [id+version_number] compound indexes, and independent parent_version_id chains. The AutosaveController emits independent save_succeeded events for each side. The "every save is preserved" guarantee is enforced by the absence of any per-version delete path — the only deletions are at the Frame or Session level, and even those preserve the historical record up to the moment of deletion. ✓
  • § 5 Human-Readable Structure. Honored. The on-disk format is JSON via Dexie; an exported FrameExport file is human-readable text. The Repository interface and the per-store layout are documented in this spec without hidden ordering tricks. ✓
  • § 6 Use-Case Commitment. Honored. The persistence layer treats Mode and Flavor as opaque schema fields that flow through unchanged; legal-mode and general-mode frames round-trip identically. Nothing in this spec privileges legal mode at the storage layer. ✓

Quality criteria (Article III §§ 1–6)

  • § 1 Correctness. Every persistence operation called for by Streams B, D, E, and H has a defined Repository method or composite operation. Every atomicity constraint from Stream D (frame-template deep-copy, session migration, restore) is wrapped in a single transaction with documented rollback semantics. Every versioning rule from Stream B (snapshots not deltas, milestone forcing, manual saves bypass debouncer) is honored. ✓
  • § 2 Legal Practice Fit. The persistence model is the engineering substrate behind a "I edit, it saves; I close the browser, my work is there" promise that lawyers expect. The version-history model — every save preserved, restore creates a new branch rather than overwriting — matches the working-document mental model from Google Docs and modern Office. The cost (multi-device pain in v1 absent cloud sync) is honestly named in Stream H and bounded by the export/import path. ✓
  • § 3 Determinism and Auditability. The clock and id injectors make every save reproducible under test. The HookInvocationRecord audit trail flows through Frame.llm_settings.invocations per Stream G. Every entity is keyed by a stable UUID. Every export is JSON that can be diffed externally. ✓

In-session flags (Article X § 1) — escalated to register

Four cross-document precision items surfaced while compiling this spec, three of them affecting the contracts doc (same family as F-003 and the I.3 pending flag) and one of them adding a test-only dependency. They consolidate into a single in-session-escalated register flag with the rationale that the contracts doc is the doc that needs the bump and downstream coding sessions are not gated on it. The four items, restated from the Cross-document resolution notes above:

  1. Persistence import boundary tightened to @/schema only. Contract 3's preamble says "persistence imports schema and runtime — the latter only to call compute on session restore." No persistence method in Stream H calls compute; recomputation happens at the state-layer subscription site after persistence returns. Tightening the boundary reinforces Article II § 2 by removing a back-door for clock-reading wrappers to thread through the runtime indirectly. Resolution: this spec commits the tighter boundary; the ESLint no-restricted-imports rule for src/persistence/ forbids @/runtime alongside the other application modules.
  2. AutosaveController takes payload-passing signatures, not (kind, id). Contract 3's schedule(kind, id) shape is not implementable within the dependency graph the contracts doc itself commits — the controller would need to reach into @/state to find the latest in-memory entity, which is forbidden (the dependency arrow runs state → persistence, not persistence → state). Resolution: this spec replaces the signature with scheduleFrameSave(payload: PendingFrameSave) / scheduleSessionSave(payload: PendingSessionSave). Multiple rapid calls for the same entity overwrite earlier pending payloads. The controller becomes a dumb storage-plus-timer abstraction; the state layer remains the source of truth for the working copy.
  3. StructuralDiff split into frame-diff and session-diff; edited entries carry fields_changed; layout-only bucketing made explicit. Contract 3's single StructuralDiff shape names nodes and edges collections, which fit frame versions but have no natural fill for session versions. Resolution: two specialized types (StructuralDiff and SessionStructuralDiff) map cleanly to Stream H's algorithm and Stream E E5's Compare view. The fields_changed: string[] extension is additive. LAYOUT_ONLY_FIELDS is exported as a constant.
