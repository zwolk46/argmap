stream_i_llm_hooks_spec_v1.html

Per-module spec for src/llm-hooks/  ·  Stream I session I.8  ·  May 11, 2026

This document is the implementation source of truth for src/llm-hooks/. It commits per-file TypeScript signatures and the concrete behavioral bodies of: the hook registry, the prompt-file format and on-disk-to-in-memory loading pipeline, the LlmProvider abstraction with its Anthropic Claude and Mock implementations, the build-time hook orchestrator (runHook / applyDecision), the SHA-256 input-hash provenance builder, and the per-hook contracts for the v1 ship list (G1–G6 + G8 + G9 + G11 + G13 with prompts; G7 + G10 + G12 with deterministic fallbacks shipped behind the same contract surface for v1.5 LLM activation). The cross-module surface (what other modules import from @/llm-hooks) is pinned in stream_i_contracts_v1.html Contract 6; this spec carries four small refinements to that contract and the neighboring Contracts 3 (persistence) and 7 (state), all logged as a single consolidated in-session-escalated flag (F-003 family) at the end.

The llm-hooks module is structurally banned from src/runtime/ (Article II § 2; ESLint no-restricted-imports in the I.1 skeleton already enforces this). Inside src/llm-hooks/ the determinism story is two-part: (a) the module performs no writes to FrameVersion or ArgumentSessionVersion fields directly — every commit is a CommitPlan the state/modes layer applies through the existing autosave path, so versioning, atomicity, and audit-trail append all flow through one place; and (b) every hook produces deterministic provenance (input hash, prompt version, model id, decision) on every commit, so any AI-influenced field is exactly attributable. The runtime is unchanged.

Out of scope (deferred to other docs or v2): the UI placement of suggestion drawers, attribution chips, and AI-Settings panels (Stream E E2 / E3; per-component design in stream_i_ui_spec_v1.html); API-key management and storage (Stream G "Items deferred"; the implementing agent picks between IndexedDB-on-app_state, browser keychain, or paste-on-each-session); the LLM bodies for v1.5 hooks G7, G10, G12 (this spec ships their HookContracts with deterministic fallbacks; the LLM bodies are written in a later coding pass); provider rate-limiting and cost telemetry; streaming completions; tool-use; learning-from-edits.

Module file map

Twenty-eight files under src/llm-hooks/ plus the prompts/ directory of markdown files. The public surface is the named re-exports from index.ts; consumers of @/llm-hooks never reach into subdirectories. Refinements to Contract 6 and neighboring contracts

This spec carries four small refinements to the cross-module surface, all logged as a single consolidated in-session-escalated flag (F-003 family) at the end. None reverses a prior stream decision; each is a precision correction or an additive surface needed for the module to compile under the disciplines the prior streams committed.

  1. Contract 6 re-exports schema-canonical types from @/schema rather than redeclaring them. The I.2 schema spec (May 10) placed LlmSettings, HookInvocationRecord, and BurdenThresholdMap canonical in @/schema/frame.ts because they are Frame and FrameVersion fields and the schema module is the single source of truth for entity shapes. Contract 6 as written in the I.1 contracts doc declares those three types as if they originated in @/llm-hooks; the schema spec's positioning is correct and Contract 6 should re-export rather than redeclare. The runtime implication is zero — TypeScript structural typing makes the surfaces identical — but the import path matters for the I.1 review session bumping the contracts doc to _v2. In this spec, @/llm-hooks/types.ts re-exports from @/schema.
  2. Repository gains loadPrompt and savePrompt methods backed by the prompts Dexie table. The I.4 persistence spec reserved the prompts store and the PromptFileRecord shape but did not expose Repository methods. The prompt-loader needs a Repository-level read path to retrieve archived prompts referenced by old HookInvocationRecords after a bundle drops them, and a write path for the loader's pass-through cache write on first bundle hit. The proposed additions: loadPrompt(hook_name: string, version: string): Promise<PromptFile | null> and savePrompt(record: PromptFileRecord): Promise<void>. The first translates a PromptFileRecord into the PromptFile shape (parsing the YAML frontmatter from record.frontmatter as an already-parsed object and exposing record.body_markdown as the body); the second writes a PromptFileRecord idempotently (using Dexie's put, so re-writes of an identical [hook_name, version] pair are no-ops).
  3. State-layer FrameStore and SessionStore gain a suggestion lifecycle. The I.5 state spec defined dispatch tables for frame and session actions but did not include hook invocations as actions. The suggestion lifecycle is a four-step state machine: idle → invoking → awaiting_decision → applying → idle. The proposed additions to Contract 7 are FrameStore.invokeHook(hook_id, args): Promise<void> (starts the orchestrator and parks the resulting SuggestionResult in a pending_suggestion slot on the store snapshot), FrameStore.resolveSuggestion(decision): Promise<void> (applies applyDecision against the parked suggestion and dispatches the resulting CommitPlan through the existing FrameActionDispatchTable as a synthetic commit_hook_output action that the modes layer adds to its table), and a pending_suggestion?: SuggestionResult<unknown> field on FrameStoreSnapshot. The same three additions apply to SessionStore for argument-layer hooks (G6, G7, G11, G12). The modes layer's FrameActionDispatchTable and SessionActionDispatchTable gain a commit_hook_output entry that translates a CommitPlan + HookInvocationRecord into a pure (entity, args) => next_entity transformation (writing the planned fields and appending the record to llm_settings.invocations).
  4. Two new v1 dependencies: yaml and mustache. The buildkit index's dependency table did not anticipate these. Both are needed for the prompt-file pipeline: yaml (Eemeli Aro's parser, ~30KB minified+gzipped) parses YAML frontmatter into a typed object; mustache (~5KB) renders the body template with the typed input. Considered and rejected: (a) a custom YAML subset parser — risky given nested-object frontmatter and the maintenance tax; (b) a custom Mustache renderer — feasible at ~50 lines but adds an in-house surface to test and maintain; (c) JSON frontmatter — diverges from Stream G's explicit "markdown with YAML frontmatter" decision. The two dependencies are the cleanest path. Neither is imported by src/runtime/ (the runtime boundary in the I.1 skeleton already pins this; ESLint no-restricted-imports enforces it). Both are tree-shake-friendly and have stable ESM exports.

The Article V § 3 trigger-recheck for these four refinements is in the self-review at the end of this document.

Determinism inside the llm-hooks module

The llm-hooks module is structurally outside the Article II § 2 runtime determinism boundary: it is never imported by src/runtime/, and its operations (LLM calls, prompt rendering, provenance hashing) inherently involve I/O. But the module still commits two practical determinism properties that the Vitest acceptance tests exercise.

  1. Sort before iterate over frame data. The ESLint argmap-determinism/no-unsorted-iteration rule, while only structurally required inside src/runtime/, is configured to apply to src/llm-hooks/ as well — per Stream G's testing strategy section. Hooks that iterate over frame.nodes, frame.edges, sibling collections, or any other unordered structure to build their TIn use the sortedIter, sortedKeys, sortedEntries helpers re-exported from @/runtime/iteration-helpers (the runtime module's helpers file is import-safe across the boundary because it is pure and has no runtime dependencies). This keeps the prompt input — and therefore the input_hash — stable across invocations regardless of incidental Map iteration order.
  2. Clock and randomness are injected via deps. buildInvocationRecord and runHook take an optional deps?: LlmHooksDeps parameter where LlmHooksDeps = { now?: () => string; generateId?: () => string; cryptoSubtle?: SubtleCrypto }; production callers omit deps (defaults read wall-clock time, generate a UUID v4 via crypto.randomUUID, and use the platform crypto.subtle); tests pass frozen-clock / fixed-id stubs. Same pattern as the layout module (I.7) and the persistence/state/modes modules (I.4/I.5/I.6).

The "deterministic enough for tests + auditable in production" property is what makes the HookInvocationRecord.input_hash field load-bearing for Article III § 3: an old FrameVersion's invocation can be replayed by loading the snapshot's prompt-version, recanonicalizing the input from the FrameVersion's structure, hashing, and comparing.

// ---- Hook contract ----

export interface HookContract<TIn, TOut> {
  id: HookId;
  name: string;
  activation: "build_time" | "output_time" | "runtime_sidecar";
  trigger: "manual" | "automatic" | "manual_with_auto_offer";
  mode_visibility: ModeVisibility;

  buildInput(ctx: HookContext): TIn;
  renderPrompt(input: TIn, prompt: PromptFile): string;
  parseOutput(raw: string, schema: JsonSchema): ParseResult<TOut>;
  fallback(input: TIn, error: ParseError | ProviderError): FallbackResult<TOut>;
  commit(output: TOut, ctx: HookContext): CommitPlan;
}

export interface HookContext {
  /** Repository handle for hooks that need it (e.g., G7 loads candidate Premises). */
  repository: Repository;
  /** The FrameVersion the hook reads. Always present; for session-layer hooks this is `session.frame_version_snapshot`. */
  frame_version: FrameVersion;
  /** Present only when the hook is argument-layer (G6, G7, G11, G12). */
  session?: ArgumentSession;
  /** The user's current focus, when relevant (e.g., the Checkpoint id for G1). */
  selection?: NodeRef | EdgeRef;
  /** Hook-specific free-form input (e.g., the pasted fact pattern for G11). */
  user_input?: unknown;
}

export type ModeVisibility = {
  legal: boolean;
  general: { personal: boolean; academic: boolean };
};

// ---- Parse / fallback envelopes ----

export type ParseResult<T> =
  | { kind: "ok"; value: T }
  | { kind: "parse_error"; error: ParseError; raw: string };

export type FallbackResult<T> =
  | { kind: "deterministic_fallback"; value: T }
  | { kind: "advise_user"; message: string }
  | { kind: "no_op" };

export class ParseError extends Error {
  readonly kind = "parse_error";
  constructor(message: string, readonly raw?: string) { super(message); }
}
export class ProviderError extends Error {
  readonly kind = "provider_error";
  constructor(message: string, readonly provider_id: string, readonly underlying?: unknown) { super(message); }
}

// ---- Commit plans ----

export interface CommitPlan {
  writes: FrameFieldWrite[];
  versioned: boolean;
}

export interface FrameFieldWrite {
  /** Which node (or edge) the write targets; absent for frame-level writes. */
  target_node_id?: NodeRef;
  target_edge_id?: EdgeRef;
  /** Dotted path within the targeted entity; "" for whole-entity replacement. */
  field_path: string;
  /** The value to write; the modes layer applies it as a deep set. */
  value: unknown;
  /** "set" replaces the field; "append" pushes into an array; "create_node" / "create_edge" allocate a fresh entity. */
  op: "set" | "append" | "create_node" | "create_edge";

Pure function splitting a prompt markdown file into its YAML frontmatter and body. The frontmatter delimiter is the line --- at the very start of the file and the next line --- after it. Files without a leading --- fence are rejected — every prompt file in v1 has a frontmatter block. The YAML parser is the yaml dependency (Eemeli Aro); we use the high-level parse function which returns plain JavaScript values for the YAML subset our frontmatter uses (strings, numbers, booleans, arrays, objects).

import { parse as parseYaml } from "yaml";

export class FrontmatterParseError extends Error {
  readonly kind = "frontmatter_parse_error";
  constructor(message: string, readonly raw?: string) { super(message); }
}

const FENCE = "---";

export function parseFrontmatter(raw: string): { frontmatter: Record<string, unknown>; body: string } {
  // Normalize line endings so the parser is OS-agnostic.
  const normalized = raw.replace(/\r\n/g, "\n");

  // First line must be exactly the fence.
  if (!normalized.startsWith(FENCE + "\n")) {
    throw new FrontmatterParseError(`prompt file is missing a leading '${FENCE}' fence`);
  }

  // Find the closing fence (a line that is exactly the fence).
  const after_first_fence = normalized.slice(FENCE.length + 1);
  const closing_index = after_first_fence.indexOf("\n" + FENCE + "\n");
  if (closing_index === -1) {
    throw new FrontmatterParseError(`prompt file is missing a closing '${FENCE}' fence`);
  }

  const yaml_block = after_first_fence.slice(0, closing_index);
  const body = after_first_fence.slice(closing_index + FENCE.length + 2);

  let frontmatter: unknown;
  try {
    frontmatter = parseYaml(yaml_block);
  } catch (err) {
    throw new FrontmatterParseError(
      `prompt frontmatter YAML did not parse: ${(err as Error).message}`,
      yaml_block,
    );
  }
  if (frontmatter === null || typeof frontmatter !== "object" || Array.isArray(frontmatter)) {
    throw new FrontmatterParseError(`prompt frontmatter must be a YAML object, not ${typeof frontmatter}`);
  }
  return { frontmatter: frontmatter as Record<string, unknown>, body };
}

// src/llm-hooks/prompt-loader.ts

The prompt-file registry. On module load, scans ./prompts/**/*.md via Vite's import.meta.glob with ?raw and eager: true, materializing every bundled prompt into an in-memory Map keyed by a compound string "<hook>@<version>". Provides one async entry point, loadPrompt, that hits the bundle first and falls back to repository.loadPrompt on miss. On the first successful bundle hit for a given [hook_name, version] pair within a session, writes through to IndexedDB so the same prompt is available for replay even if a future bundle drops it. The replay case is the constitutional load-bearing path: an old FrameVersion's HookInvocationRecord may reference a prompt version the live bundle no longer ships, and Article III § 3 (auditability) requires that prompt remain retrievable.

The preloadBundledPrompts function is called from the RepositoryProvider's mount effect (the I.5 state spec already mounts the repository at the React app root; an I.5 review session adds the preload call after I.8 lands) so that the IDB cache is warm before the first hook invocation. Tests bypass this by constructing the loader against a synthetic bundle.

import type { PromptFile } from "./types";
import type { Repository } from "@/persistence";
import { parseFrontmatter } from "./prompt-frontmatter";

export class PromptNotFoundError extends Error {
  readonly kind = "prompt_not_found";
  constructor(readonly hook_name: string, readonly version: string) {
    super(`prompt not found: ${hook_name}@${version}`);
  }
}

// `import.meta.glob` is evaluated at build time by Vite. The keys are file
// paths (e.g., "./prompts/g1-checkpoint-question/v1.md"); the values are the
// raw markdown strings (because of `query: '?raw', import: 'default', eager: true`).
const BUNDLED_RAW: Record<string, string> = import.meta.glob(
  "./prompts/**/*.md",
  { query: "?raw", import: "default", eager: true },
) as Record<string, string>;

// Parse the bundle into a typed map at module load. Iteration order is
// deterministic (Vite's import.meta.glob results sort lexicographically by path
// when iterated; we sort defensively).
const BUNDLED: Map<string, PromptFile> = (() => {
  const map = new Map<string, PromptFile>();
  const path_re = /^\.\/prompts\/([^/]+)\/([^/]+)\.md$/;
  for (const path of Object.keys(BUNDLED_RAW).sort()) {
    const match = path_re.exec(path);
    if (!match) continue;
    const [, hook_name, version] = match;
    const raw = BUNDLED_RAW[path];
    const { frontmatter, body } = parseFrontmatter(raw);
    const file: PromptFile = {
      hook_name: String(frontmatter.hook_name ?? hook_name),
      version: String(frontmatter.version ?? version),
      schema_in: frontmatter.schema_in ?? {},
      schema_out: frontmatter.schema_out ?? {},
      model_hint: frontmatter.model_hint as string | undefined,
      provider_hints: frontmatter.provider_hints as Record<string, unknown> | undefined,
      body,
      created_at: frontmatter.created_at as string | undefined,
      notes: frontmatter.notes as string | undefined,
    };
    // The path-derived hook_name/version are the source of truth; a build-time
    // test asserts they always match the frontmatter values.
    if (file.hook_name !== hook_name || file.version !== version) continue;
    map.set(`${hook_name}@${version}`, file);
  }
  return map;
})();

/**
 * Resolve a prompt by hook_name + version. Hits the bundle first; falls back to
 * the IndexedDB `prompts` store for archived prompts no longer in the bundle.
 * On a bundle hit, writes through to IDB idempotently for replay availability.
 */
export async function loadPrompt(
  hook_name: string,
  version: string,
  repository: Repository,
): Promise<PromptFile> {
  const key = `${hook_name}@${version}`;
  const bundled = BUNDLED.get(key);
  if (bundled) {
    // Write-through to IDB for replay availability. Repository.savePrompt
    // uses Dexie `put`, so re-writes are no-ops.
    await repository.savePrompt({
      hook_name,
      version,
      body_markdown: bundled.body,
      frontmatter: extractFrontmatterObject(bundled),
      added_at: bundled.created_at ?? new Date(0).toISOString(),
    });
    return bundled;
  }
  const archived = await repository.loadPrompt(hook_name, version);
  if (archived) return archived;
  throw new PromptNotFoundError(hook_name, version);
}

/**
 * Eager-write every bundled prompt to IDB. Called once at app start.
 * Idempotent.
 */
export async function preloadBundledPrompts(repository: Repository): Promise<void> {
  for (const [, file] of BUNDLED) {
    await repository.savePrompt({
      hook_name: file.hook_name,
      version: file.version,
      body_markdown: file.body,
      frontmatter: extractFrontmatterObject(file),
      added_at: file.created_at ?? new Date(0).toISOString(),
    });
  }
}

function extractFrontmatterObject(file: PromptFile): Record<string, unknown> {
  return {
    hook_name: file.hook_name,
    version: file.version,
    schema_in: file.schema_in,
    schema_out: file.schema_out,
    model_hint: file.model_hint,
    provider_hints: file.provider_hints,
    created_at: file.created_at,
    notes: file.notes,
  };
}

// Test seam: tests construct an alternate loader against a synthetic bundle.
export function _makeLoaderForTests(synthetic_bundle: Map<string, PromptFile>) {
  return async function load(hook_name: string, version: string, repository: Repository): Promise<PromptFile> {
    const key = `${hook_name}@${version}`;
    const hit = synthetic_bundle.get(key);
    if (hit) return hit;
    const archived = await repository.loadPrompt(hook_name, version);
    if (archived) return archived;
    throw new PromptNotFoundError(hook_name, version);
  };
}

  CompletionResponse,
  ProviderCapabilities,

export class AnthropicProvider implements LlmProvider {
  readonly id = "anthropic";
  readonly capabilities: ProviderCapabilities = {
    structured_output: false,
    streaming: false,
    tool_use: false,
  };
  private readonly client: Anthropic;
  private readonly default_model: string;

  constructor(config: AnthropicProviderConfig) {
    this.client = new Anthropic({ apiKey: config.api_key, baseURL: config.base_url });
    this.default_model = config.default_model ?? DEFAULT_ANTHROPIC_MODEL;

  async complete(req: CompletionRequest): Promise<CompletionResponse> {
    const model = req.model_hint ?? this.default_model;
    try {

        model,
        max_tokens: req.max_tokens ?? 1024,
        temperature: req.temperature ?? 0.2,
        messages: [{ role: "user", content: req.prompt }],

        .filter((block): block is Anthropic.TextBlock => block.type === "text")
        .map((block) => block.text)
        .join("");
      return {
        raw_text,
        model_id: response.model,
        finish_reason: response.stop_reason === "end_turn" ? "stop"
                     : response.stop_reason === "max_tokens" ? "length"
                     : "stop",
        token_usage: {
          input: response.usage.input_tokens,
          output: response.usage.output_tokens,
        },
      };
    } catch (err) {
      throw new ProviderError(
        `Anthropic provider error: ${(err as Error).message}`,
        "anthropic",
        err,
      );
    }
  }
}

Pure functions for provenance. canonicalize produces a sort-stable JSON serialization (object keys sorted lexicographically at every depth; arrays preserve order; undefined values are dropped; NaN and infinities are forbidden). hashCanonical returns the SHA-256 of the canonical bytes as lowercase hex. buildInvocationRecord assembles a HookInvocationRecord from a SuggestionResult + a ConfirmationDecision + the commit plan derived for that decision. The clock and id sources are injectable via ProvenanceDeps.

import type {
  HookInvocationRecord,
  SuggestionResult,
  ConfirmationDecision,
  CommitPlan,
  LlmHooksDeps,
} from "./types";
import type { NodeRef } from "@/schema";

export type ProvenanceDeps = LlmHooksDeps;

/**
 * Sort-stable JSON serialization. Object keys are sorted lexicographically at
 * every depth; arrays preserve order.
 */
export function canonicalize(input: unknown): string {
  return JSON.stringify(input, function (this: unknown, _key: string, value: unknown) {
    if (value === undefined) return undefined;
    if (typeof value === "number" && !Number.isFinite(value)) {
      throw new Error("canonicalize: non-finite numbers are not allowed");
    }
    if (value && typeof value === "object" && !Array.isArray(value)) {
      const obj = value as Record<string, unknown>;
      const sorted: Record<string, unknown> = {};
      for (const k of Object.keys(obj).sort()) sorted[k] = obj[k];
      return sorted;
    }
    return value;
  });
}

/** SHA-256 of a canonicalized string as lowercase hex. */
export async function hashCanonical(canonical: string, deps: ProvenanceDeps = {}): Promise<string> {
  const subtle = deps.cryptoSubtle ?? globalThis.crypto.subtle;
  const bytes = new TextEncoder().encode(canonical);
  const buf = await subtle.digest("SHA-256", bytes);
  const view = new Uint8Array(buf);
  let hex = "";
  for (let i = 0; i < view.length; i++) {
    hex += view[i].toString(16).padStart(2, "0");
  }
  return hex;
}

export interface BuildInvocationArgs {
  suggestion: SuggestionResult<unknown>;
  decision: ConfirmationDecision<unknown>;
  commit_plan: CommitPlan;
  /** True when the modes layer is about to write the plan; false on rejection. */
  committed: boolean;
}

export function buildInvocationRecord(
  args: BuildInvocationArgs,
  deps: ProvenanceDeps = {},
): HookInvocationRecord {
  const now = deps.now ? deps.now() : new Date().toISOString();
  const id = deps.generateId ? deps.generateId() : globalThis.crypto.randomUUID();

  const { suggestion, decision, commit_plan, committed } = args;
  const target_node_ids: NodeRef[] = [];
  const target_field_paths: string[] = [];
  for (const write of commit_plan.writes) {
    if (write.target_node_id) target_node_ids.push(write.target_node_id);
    target_field_paths.push(
      (write.target_node_id ? `${write.target_node_id}:` : "") + write.field_path,
    );
  }
  target_node_ids.sort();
  target_field_paths.sort();

  const decision_kind = decision.kind;
  const final_value = decision_kind === "rejected" ? undefined : decision.final;
  return {
    id,
    hook_id: suggestion.hook_id,
    prompt_name: suggestion.prompt_name,
    prompt_version: suggestion.prompt_version,
    provider_id: suggestion.provider_id,
    model_id: suggestion.model_id,
    input_hash: suggestion.input_hash,
    raw_response: suggestion.raw_response,
    decision: decision_kind,
    final_value,
    target_node_ids: target_node_ids.length ? target_node_ids : undefined,
    target_field_paths: target_field_paths.length ? target_field_paths : undefined,
    invoked_at: suggestion.generated_at,
    committed_at: committed && decision_kind !== "rejected" ? now : undefined,
  };
}

The orchestrator. runHook wires the hook's buildInput → prompt-loader → hook's renderPrompt → provider's complete → hook's parseOutput (with the fallback path on parse or provider error), returning a typed SuggestionResult. applyDecision turns a user's accept/edit/reject decision into a CommitPlan + HookInvocationRecord. The orchestrator is pure with respect to frame/session state: it never calls repository.save* directly. The state layer's action runner is responsible for persisting the plan and the record atomically.

import type {
  HookContract,
  HookContext,
  LlmProvider,
  SuggestionResult,
  ConfirmationDecision,
  CommitPlan,
  HookInvocationRecord,
  LlmHooksDeps,
  ParseResult,
  FallbackResult,
} from "./types";
import { ProviderError } from "./types";
import { loadPrompt } from "./prompt-loader";
import { canonicalize, hashCanonical, buildInvocationRecord } from "./provenance";

export interface RunHookOptions {
  /**
   * The prompt version to load (per `LlmSettings.prompt_versions[hook.name]`).
   * Production callers pass `frame.llm_settings?.prompt_versions?.[hook.name] ?? "v1"`.
   */
  prompt_version: string;
  max_tokens?: number;
  temperature?: number;
}

export async function runHook<TIn, TOut>(
  hook: HookContract<TIn, TOut>,
  context: HookContext,
  provider: LlmProvider,
  options: RunHookOptions,
  deps: LlmHooksDeps = {},
): Promise<SuggestionResult<TOut>> {
  const now = deps.now ? deps.now() : new Date().toISOString();
  const input = hook.buildInput(context);
  const canonical_input = canonicalize(input);
  const input_hash = await hashCanonical(canonical_input, deps);

  const prompt_file = await loadPrompt(hook.name, options.prompt_version, context.repository);
  const rendered = hook.renderPrompt(input, prompt_file);

  let raw_text = "";
  let model_id = provider.id;
  let parsed: TOut | undefined;
  let parse_status: "ok" | "fallback" | "error" = "ok";

  try {
    const response = await provider.complete({
      prompt: rendered,
      output_schema: prompt_file.schema_out,
      model_hint: prompt_file.model_hint,
      max_tokens: options.max_tokens,
      temperature: options.temperature,
    });
    raw_text = response.raw_text;
    model_id = response.model_id;
    const parse_result: ParseResult<TOut> = hook.parseOutput(raw_text, prompt_file.schema_out);
    if (parse_result.kind === "ok") {
      parsed = parse_result.value;
    } else {
      const fallback = hook.fallback(input, parse_result.error);
      ({ parsed, parse_status } = projectFallback(fallback, "fallback"));
    }
  } catch (err) {
    const provider_err = err instanceof ProviderError
      ? err
      : new ProviderError(`unknown provider error: ${(err as Error).message}`, provider.id, err);
    const fallback = hook.fallback(input, provider_err);
    ({ parsed, parse_status } = projectFallback(fallback, "error"));
  }

  return {
    hook_id: hook.id,
    prompt_name: prompt_file.hook_name,
    prompt_version: prompt_file.version,
    provider_id: provider.id,
    model_id,
    input_hash,
    raw_response: raw_text,
    parsed: parsed as TOut,                  // safe: projectFallback always assigns
    parse_status,
    generated_at: now,
    echo_input: input,
  };
}

function projectFallback<T>(
  fallback: FallbackResult<T>,
  status_on_value_kind: "fallback" | "error",
): { parsed: T; parse_status: "ok" | "fallback" | "error" } {
  if (fallback.kind === "deterministic_fallback") {
    return { parsed: fallback.value, parse_status: "fallback" };
  }
  // advise_user and no_op: synthesize an empty value so the UI can surface the
  // SuggestionResult with parse_status === "error". The UI's drawer checks
  // parse_status before rendering; "error" surfaces the message in raw_response
  // (which the projectFallback caller has already populated to a useful string
  // via the fallback's advise_user.message when available; the runHook caller
  // assigns this conditionally).
  return { parsed: undefined as unknown as T, parse_status: status_on_value_kind };
}

/**
 * Translate a user decision into the commit plan and invocation record the
 * state/modes layer applies. Pure with respect to frame state.
 */
export function applyDecision<TOut>(
  hook: HookContract<unknown, TOut>,
  suggestion: SuggestionResult<TOut>,
  decision: ConfirmationDecision<TOut>,
  context: HookContext,
  deps: LlmHooksDeps = {},
): { commit_plan: CommitPlan; invocation_record: HookInvocationRecord } {
  let commit_plan: CommitPlan;
  if (decision.kind === "rejected") {
    commit_plan = { writes: [], versioned: false };
  } else {
    commit_plan = hook.commit(decision.final, context);
  }
  const invocation_record = buildInvocationRecord(
    {
      suggestion: suggestion as SuggestionResult<unknown>,
      decision: decision as ConfirmationDecision<unknown>,
      commit_plan,
      committed: decision.kind !== "rejected",
    },
    deps,
  );
  return { commit_plan, invocation_record };
}

Per-hook files

Each hooks/<name>.ts exports a single HookContract handle. The hook's behavior — input shape, output shape, parser rules, fallback behavior, commit semantics — is fully specified in stream_g_llm_hooks_v1.html's per-hook cards; this section pins the file-level TypeScript signatures only. Coding sessions implement against the Stream G cards plus the signatures here, then write src/llm-hooks/README.md documenting any per-hook choices that downstream sessions need to know.

Two structural points common to every hook file:

  • Pure with respect to state. The hook's commit function returns a CommitPlan; it never calls repository.save*. The state layer's action runner is the single caller responsible for persisting the plan (including the append to llm_settings.invocations) in a Dexie transaction per the F-003-family Repository extension noted above.
  • Sorted iteration over frame data. Self-review under Article VIII § 4

Architectural commitments (Article II)

  • § 1 Frame/Argument Split. Honored. Hooks operate on frame-layer entities (G1–G5, G8, G9, G13) or argument-layer entities (G6, G7, G11, G12) via the HookContext.frame_version / HookContext.session separation. No hook commits to both layers in a single CommitPlan; the modes layer's commit_hook_output action runner routes the plan to the appropriate dispatch table (frame or session) per the hook's activation field.
  • § 2 Determinism. Honored. The llm-hooks module is structurally banned from src/runtime/ by the I.1 skeleton's ESLint no-restricted-imports rule. The module's two practical-determinism properties (sorted iteration over frame data; clock/id/subtle injection via LlmHooksDeps) are exercised by the iteration-order test, the canonicalize-stability test, and the per-hook round-trip-determinism tests. The runtime's compute pipeline reads only fixed FrameVersion fields and llm_settings_snapshot.calibrated_thresholds (F-001 projection); no hook output reaches compute() through any other path. Hook commits go through the modes layer's existing FrameVersion / ArgumentSessionVersion versioning, so versioning, atomicity, and audit-trail append all flow through one place.
  • § 3 Per-Node Satisfaction Criteria. Honored. G5's calibrated thresholds influence burden_met evaluation via the F-001 projection only; no hook rewrites SatisfactionPolicy or alters the 12-condition library. The conditions library remains the schema's territory.
  • § 4 Independent Versioning. Honored. Frame-layer hook commits create a new FrameVersion; argument-layer hook commits (G6, G11) create a new ArgumentSessionVersion; G4 / G7 / G10 / G12 do not version anything. The modes layer routes the appropriate versioned: true/false field on the CommitPlan to the matching autosave path per the I.6 modes spec.
  • § 5 Human-Readable Structure. Honored. Hook outputs commit to existing schema fields (Checkpoint.question, Conclusion.reasoning_summary, FrameVersion.change_summary, etc.); a reader of the JSON envelope sees plain content with no AI-special encoding.   • § 6 Use-Case Commitment. Honored. Quality criteria (Article III)

  • § 1 Correctness. Each hook is specified against a known surface (Stream G card) with a typed input drawn from @/schema's entity shapes, a typed output that maps to a defined frame/session field or to a presentational artifact, and a parser whose rules are implementable from Stream G's stated heuristics.   • § 2 Legal Practice Fit. The legal-flavored hooks (G1's jurisdiction context, G2's citation_hint, G5's standard-of-review calibration, G8's legal direction enum) are scoped to surfaces where AI assistance is plausibly useful to a practicing lawyer; the parser constraints (G2's "no fabricated citations" rule, G3's "no future-tense legal prediction" rule, G11's "source quote must be substring" rule) reflect known LLM failure modes that lawyers are increasingly sensitive to.
  • § 3 Determinism and Auditability. Maximally satisfied. The structural import boundary is checkable in CI. The per-hook bake-and-commit discipline is the dominant pattern. The HookInvocationRecord.input_hash + raw_response + prompt_version + model_id tuple lets an old FrameVersion's invocation be exactly replayed.
  • § 4 Clarity. The contract is one interface implemented thirteen times; the prompt-file format is one schema with one parsing path; the registry has one lookup function. Open the Frame's llm_settings.invocations array and you see every AI commit ever made on the frame.
  • § 5 Reversibility. Locked-in choices: the HookContract interface, the PromptFile schema, the LlmProvider interface, the LlmSettings shape, the bake-and-commit discipline.
